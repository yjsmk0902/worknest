# Worknest 인증 화면 설계

> 이 문서는 로그인, 회원가입, 초대 수락, 온보딩 위저드 화면의 레이아웃과 컴포넌트 스펙을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 공통 레이아웃

모든 인증 화면은 동일한 셸(Shell) 레이아웃을 사용합니다.

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│                                                                  │
│                                                                  │
│                    ┌─────────────────────┐                       │
│                    │                     │                       │
│                    │   [Worknest 로고]    │                       │
│                    │                     │                       │
│                    │   ┌─── 카드 ──────┐ │                       │
│                    │   │               │ │                       │
│                    │   │  (폼 콘텐츠)   │ │                       │
│                    │   │               │ │                       │
│                    │   └───────────────┘ │                       │
│                    │                     │                       │
│                    │   [하단 링크]        │                       │
│                    │                     │                       │
│                    └─────────────────────┘                       │
│                                                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 레이아웃 스펙

| 속성 | 값 |
|------|-----|
| 배경 | `bg-background` |
| 컨테이너 정렬 | `flex items-center justify-center min-h-screen` |
| 카드 너비 | **400px** (max) |
| 카드 패딩 | `p-6` (24px) |
| 카드 배경 | `bg-card` |
| 카드 보더 | `border border-border rounded-lg` |
| 카드 그림자 | `shadow-sm` |
| 로고 위치 | 카드 상단, 중앙 정렬, `mb-8` |
| 폼 간격 | 필드 간 `space-y-4` (16px) |
| 하단 링크 | 카드 아래 `mt-4`, 중앙 정렬 |

## 2. 로그인 페이지 (`/login`)

### 2.1 기본 상태

```
┌─────────────────────────────────────────┐
│                                         │
│           [Worknest 로고 + 텍스트]        │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │                                   │  │
│  │  로그인                            │  │  ← text-2xl font-semibold
│  │  워크스페이스에 로그인하세요           │  │  ← text-sm text-muted-foreground
│  │                                   │  │
│  │  이메일                            │  │  ← label text-sm font-medium
│  │  ┌───────────────────────────┐    │  │
│  │  │ name@company.com          │    │  │  ← input, placeholder
│  │  └───────────────────────────┘    │  │
│  │                                   │  │
│  │  비밀번호                          │  │
│  │  ┌───────────────────────────┐    │  │
│  │  │ ••••••••            [👁]  │    │  │  ← password + toggle
│  │  └───────────────────────────┘    │  │
│  │                                   │  │
│  │  ┌───────────────────────────┐    │  │
│  │  │        로그인              │    │  │  ← primary 버튼, w-full
│  │  └───────────────────────────┘    │  │
│  │                                   │  │
│  └───────────────────────────────────┘  │
│                                         │
│     계정이 없으신가요? 회원가입            │  ← text-sm, "회원가입" = 링크
│                                         │
└─────────────────────────────────────────┘
```

### 2.2 컴포넌트 스펙

#### 이메일 입력 필드

| 속성 | 값 |
|------|-----|
| type | `email` |
| placeholder | `name@company.com` |
| 자동완성 | `autocomplete="email"` |
| 검증 | 이메일 형식 (Zod: `z.string().email()`) |

#### 비밀번호 입력 필드

| 속성 | 값 |
|------|-----|
| type | `password` (토글로 `text` 전환) |
| placeholder | 없음 |
| 자동완성 | `autocomplete="current-password"` |
| 최소 길이 | 8자 |
| 토글 아이콘 | `Eye` / `EyeOff` (Lucide, 16px) |

#### 로그인 버튼

| 속성 | 값 |
|------|-----|
| 스타일 | `variant="default"` (primary 배경) |
| 너비 | `w-full` |
| 크기 | `h-10` (40px) |
| 텍스트 | `로그인` |
| 단축키 | Enter (폼 제출) |

### 2.3 상태별 변형

#### 유효성 검증 에러

```
  이메일
  ┌───────────────────────────┐
  │ invalid-email             │  ← border-destructive
  └───────────────────────────┘
  올바른 이메일 주소를 입력해주세요.    ← text-sm text-destructive, mt-1

  비밀번호
  ┌───────────────────────────┐
  │                      [👁] │  ← border-destructive
  └───────────────────────────┘
  비밀번호는 8자 이상이어야 합니다.     ← text-sm text-destructive, mt-1
```

- 입력 필드 border: `border-destructive`
- 에러 메시지: 필드 바로 아래, `text-sm text-destructive`
- 포커스 링: `ring-destructive`

#### 로그인 실패 (잘못된 자격증명)

```
  ┌─────────────────────────────────────┐
  │ ⚠ 이메일 또는 비밀번호가 올바르지       │  ← bg-destructive/10
  │   않습니다.                          │     text-destructive
  └─────────────────────────────────────┘     border-destructive/20
                                              rounded-md p-3 mb-4
  이메일
  ┌───────────────────────────┐
  │ user@company.com          │
  └───────────────────────────┘
```

- 알림 배너: 폼 상단, `bg-destructive/10` 배경에 `text-destructive` 텍스트
- 아이콘: `AlertTriangle` (Lucide, 16px)

#### 로딩 상태

```
  ┌───────────────────────────┐
  │    [스피너] 로그인 중...     │  ← 버튼 disabled, opacity-70
  └───────────────────────────┘
```

- 버튼 텍스트: `로그인 중...`
- 스피너: `Loader2` 아이콘 (Lucide), `animate-spin`, 버튼 텍스트 앞
- 버튼 상태: `disabled`, `opacity-70`, 클릭 불가
- 입력 필드: `disabled` (편집 불가)

#### Rate Limit 메시지

```
  ┌─────────────────────────────────────┐
  │ ⚠ 로그인 시도가 너무 많습니다.         │  ← bg-destructive/10
  │   5분 후에 다시 시도해주세요.           │
  └─────────────────────────────────────┘
```

- 5회 실패 후 표시
- 5분 잠금 동안 폼은 `disabled` 상태
- 카운트다운 타이머 표시 고려 (선택사항)

### 2.4 키보드 인터랙션

| 키 | 동작 |
|-----|------|
| Tab | 이메일 → 비밀번호 → 비밀번호 토글 → 로그인 버튼 → 회원가입 링크 |
| Enter | 폼 제출 (어느 필드에서든) |
| Space | 비밀번호 토글 버튼 활성화 |

### 2.5 접근성

- 폼: `<form aria-label="로그인">`
- 에러 메시지: `aria-describedby`로 입력 필드와 연결
- 알림 배너: `role="alert"`
- 비밀번호 토글: `aria-label="비밀번호 표시"` / `aria-label="비밀번호 숨기기"`

### 3.2 컴포넌트 스펙

#### 이름 입력 필드

| 속성 | 값 |
|------|-----|
| type | `text` |
| placeholder | `홍길동` |
| 자동완성 | `autocomplete="name"` |
| 검증 | 1~50자 (Zod: `z.string().min(1).max(50)`) |

#### 이메일 입력 필드

| 속성 | 값 |
|------|-----|
| type | `email` |
| placeholder | `name@company.com` |
| 자동완성 | `autocomplete="email"` |
| 검증 | 이메일 형식 |

#### 비밀번호 입력 필드

| 속성 | 값 |
|------|-----|
| type | `password` |
| 자동완성 | `autocomplete="new-password"` |
| 최소 길이 | 8자 |
| 도움말 | `8자 이상, 영문/숫자 포함` — `text-xs text-muted-foreground` |
| 토글 아이콘 | `Eye` / `EyeOff` |

### 3.3 상태별 변형

#### 유효성 검증 에러

```
  이름
  ┌───────────────────────────┐
  │                           │  ← border-destructive
  └───────────────────────────┘

  이메일
  ┌───────────────────────────┐
  │ invalid                   │  ← border-destructive
  └───────────────────────────┘
  올바른 이메일 주소를 입력해주세요.

  비밀번호
  ┌───────────────────────────┐
  │ •••                  [👁] │  ← border-destructive
  └───────────────────────────┘
  비밀번호는 8자 이상이어야 합니다.
```

#### 이메일 중복 에러

```
  ┌─────────────────────────────────────┐
  │ ⚠ 이미 사용 중인 이메일입니다.         │
  │   로그인 페이지로 이동하시겠습니까?     │  ← "로그인 페이지" = 링크
  └─────────────────────────────────────┘
```

#### 로딩 상태

```
  ┌───────────────────────────┐
  │   [스피너] 계정 생성 중...   │  ← disabled, opacity-70
  └───────────────────────────┘
```

#### 가입 성공

- 가입 완료 시: 온보딩 위저드(`/onboarding`)로 자동 리다이렉트
- 초대를 통한 가입 시: 해당 워크스페이스로 자동 리다이렉트

### 3.4 키보드 인터랙션

| 키 | 동작 |
|-----|------|
| Tab | 이름 → 이메일 → 비밀번호 → 비밀번호 토글 → 회원가입 버튼 → 로그인 링크 |
| Enter | 폼 제출 |

### 5.8 이미 로그인된 사용자가 접근 시

- Organization이 이미 있는 사용자가 `/onboarding` 접근: 워크스페이스로 리다이렉트
- 로그인하지 않은 사용자가 접근: `/login`으로 리다이렉트

## 6. 인증 라우팅 규칙 요약

```
비로그인 + 보호된 페이지 → /login (returnUrl 쿼리 파라미터 보존)
로그인 + /login 또는 /register → /:orgSlug/:wsSlug (마지막 워크스페이스)
로그인 + Organization 없음 → /onboarding
로그인 + Workspace 없음 → /orgs (조직 목록)
초대 수락 완료 → 해당 워크스페이스
가입 완료 → /onboarding
```

### returnUrl 처리

```
1. 비로그인 사용자가 /acme/dev/projects/WORK/issues 접근 시도
2. /login?returnUrl=%2Facme%2Fdev%2Fprojects%2FWORK%2Fissues 로 리다이렉트
3. 로그인 성공 시 returnUrl로 복귀
4. returnUrl이 없으면 마지막 워크스페이스로 이동
```

## 7. 공통 폼 컴포넌트 스펙

### 7.1 Input

```
┌─ 기본 ──────────────────────────┐
│  라벨                text-sm font-medium mb-1.5
│  ┌─────────────────────────┐
│  │ 플레이스홀더              │    h-10, px-3, text-sm

│  도움말 텍스트                    text-xs text-muted-foreground mt-1

┌─ 에러 ──────────────────────────┐
│  라벨
│  ┌─────────────────────────┐
│  │ 입력값                   │    border-destructive
│  └─────────────────────────┘    focus: ring-destructive
│  에러 메시지                      text-sm text-destructive mt-1
└─────────────────────────────────┘

┌─ 비활성 ────────────────────────┐
│  라벨
│  ┌─────────────────────────┐
│  │ 값                      │    bg-muted, cursor-not-allowed
│  └─────────────────────────┘    opacity-50
└─────────────────────────────────┘
```

| 상태 | 보더 | 배경 | 포커스 |
|------|------|------|--------|
| 기본 | `border-input` | `bg-background` | `ring-2 ring-ring` |
| 호버 | `border-input` (변화 없음) | `bg-background` | — |
| 포커스 | `border-input` | `bg-background` | `ring-2 ring-ring ring-offset-2` |
| 에러 | `border-destructive` | `bg-background` | `ring-2 ring-destructive` |
| 비활성 | `border-input` | `bg-muted` | 없음 |

### 7.2 Button 변형

| 변형 | 배경 | 텍스트 | 보더 | 용도 |
|------|------|--------|------|------|
| default (primary) | `bg-primary` | `text-primary-foreground` | 없음 | 주요 액션 |
| secondary | `bg-secondary` | `text-secondary-foreground` | 없음 | 보조 액션 |
| outline | `bg-background` | `text-foreground` | `border-input` | 취소, 이전 |
