# Worknest 벌크 액션 설계 (CP3-DS-4)

> 이 문서는 이슈 벌크 선택, 선택 바(Selection Bar), 벌크 속성 변경, 벌크 삭제의 레이아웃과 인터랙션을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

벌크 액션은 여러 이슈를 선택하여 상태, 우선순위, 담당자, 라벨을 일괄 변경하거나 삭제하는 기능입니다. 리스트 뷰에서만 지원됩니다 (보드 뷰는 단일 카드 DnD만 MVP).

### 1.1 핵심 원칙

- **명확한 선택 표시**: 선택된 행은 배경색 + 체크박스로 명확히 구분
- **빠른 접근**: 선택 즉시 하단 액션 바가 나타나 즉시 조작 가능
- **안전 장치**: 삭제는 확인 Dialog 필수, 변경은 확인 없이 즉시 적용
- **제한**: 최대 50건까지 선택 가능 (서버 벌크 API 제한과 동일)

## 2. 선택 메커니즘

### 2.1 선택 방법

| 방법 | 동작 |
|------|------|
| 체크박스 클릭 | 해당 행 선택/해제 토글 |
| `X` 키 | 포커스된 행의 선택 토글 (J/K 탐색과 연계) |
| 헤더 체크박스 | 전체 선택/해제 토글 |
| `Shift + 클릭` | 마지막 선택 행 ~ 클릭 행 범위 선택 |

### 2.2 선택 상태 시각화

```
┌─ 선택된 행 ─────────────────────────────────────────────────────────────┐
│ ☑  ● High   WRK-42   로그인 시 비밀번호 에러 메시지 미표시   ● In Prog  👤 │
│ bg-primary/5                                                           │
│ checkbox: checked (primary color)                                      │
└────────────────────────────────────────────────────────────────────────┘

┌─ 미선택 행 ─────────────────────────────────────────────────────────────┐
│ ☐  ● Med    WRK-43   회원가입 API 구현                     ● Todo     👤 │
│ bg-background                                                          │
│ checkbox: unchecked                                                    │
└────────────────────────────────────────────────────────────────────────┘
```

| 상태 | 배경 | 체크박스 |
|------|------|---------|
| 미선택 | `bg-background` | unchecked |
| 선택됨 | `bg-primary/5` | checked (`border-primary bg-primary`) |
| 호버 + 미선택 | `bg-accent/50` | unchecked |
| 호버 + 선택됨 | `bg-primary/5` (변화 없음) | checked |

### 2.3 헤더 체크박스

```
┌─ 헤더 ──────────────────────────────────────────────────────────────────┐
│ ☐  Priority   Key        Title                    Status  Assignee  Due│
│ ↑                                                                      │
│ 전체 선택/해제 토글                                                      │
└────────────────────────────────────────────────────────────────────────┘
```

| 상태 | 체크박스 표시 | 동작 |
|------|-------------|------|
| 0건 선택 | ☐ (비어있음) | 클릭 → 현재 페이지 전체 선택 |
| 일부 선택 | ☑ (반 체크, `indeterminate`) | 클릭 → 전체 해제 |
| 전체 선택 | ☑ (전체 체크) | 클릭 → 전체 해제 |

### 2.4 50건 제한

| 상황 | 동작 |
|------|------|
| 50건 선택 시도 | 50번째 이후 체크박스 비활성(`disabled`), Tooltip: `"최대 50건까지 선택 가능"` |
| 헤더 체크박스 (50건 초과 목록) | 처음 50건만 선택, Tooltip: `"최대 50건까지 선택 가능"` |

## 3. 선택 바 (Selection Bar)

### 3.1 레이아웃

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  3건 선택됨     [상태 변경 ▼] [우선순위 변경 ▼] [담당자 변경] [라벨 변경]     │  ← 고정 하단
│                                                        [전체 해제] [삭제]  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 선택 바 스타일

| 속성 | 값 |
|------|-----|
| 위치 | `fixed bottom-0 left-0 right-0` (사이드바 너비만큼 `left` 조정) |
| 높이 | `h-12` (48px) |
| 배경 | `bg-background` |
| 상단 보더 | `border-t border-border` |
| 그림자 | `shadow-lg` (위쪽으로 그림자) |
| z-index | `z-50` |
| 레이아웃 | `flex items-center justify-between px-4` |
| 진입 애니메이션 | `translate-y-full → translate-y-0`, `duration-150 ease-out` |
| 퇴장 애니메이션 | `translate-y-0 → translate-y-full`, `duration-100 ease-in` |
| 표시 조건 | 1건 이상 선택 시 |

### 3.3 왼쪽 영역: 선택 카운트

| 속성 | 값 |
|------|-----|
| 텍스트 | `"{N}건 선택됨"`, `text-sm font-medium` |
| 예시 | `"3건 선택됨"`, `"50건 선택됨"` |

### 3.4 중앙 영역: 액션 버튼

| 버튼 | 컴포넌트 | 아이콘 | 동작 |
|------|---------|--------|------|
| 상태 변경 | `Button` variant=`outline` size=`sm` | 없음 | `DropdownMenu` 열기 |
| 우선순위 변경 | `Button` variant=`outline` size=`sm` | 없음 | `DropdownMenu` 열기 |
| 담당자 변경 | `Button` variant=`outline` size=`sm` | 없음 | `Popover` 열기 |
| 라벨 변경 | `Button` variant=`outline` size=`sm` | 없음 | `Popover` 열기 |

### 3.5 오른쪽 영역: 해제 + 삭제

| 버튼 | 스타일 | 동작 |
|------|--------|------|
| 전체 해제 | `Button` variant=`ghost` size=`sm`, `text-muted-foreground` | 모든 선택 해제 |
| 삭제 | `Button` variant=`destructive` size=`sm` | 확인 Dialog 열기 |

## 4. 벌크 속성 변경

### 4.1 상태 변경 DropdownMenu

```
[상태 변경 ▼] 클릭
      │
      └─ DropdownMenu 열림
         ┌──────────────────────┐
         │ ● Backlog            │  ← 상태 목록
         │ ● Todo               │     카테고리 색상 dot + 상태명
         │ ● In Progress        │     text-sm
         │ ● Done               │     hover: bg-accent
         │ ● Cancelled          │
         └──────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `DropdownMenu` (shadcn/ui) |
| 항목 | `flex items-center gap-2`, 색상 dot `w-2.5 h-2.5 rounded-full` + 상태명 `text-sm` |
| 선택 | 클릭 즉시 선택된 모든 이슈의 상태 변경 (낙관적 업데이트) |
| 혼합 상태 | 현재 선택에 체크 표시 없음 (벌크이므로 단일 값을 설정) |

### 4.2 우선순위 변경 DropdownMenu

```
[우선순위 변경 ▼] 클릭
      │
      └─ DropdownMenu 열림
         ┌──────────────────────┐
         │ 🔴 Urgent            │
         │ 🟠 High              │
         │ 🟡 Medium            │
         │ 🔵 Low               │
         │ ⚪ None               │
         └──────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `DropdownMenu` (shadcn/ui) |
| 항목 | `flex items-center gap-2`, 색상 dot `w-3 h-3 rounded-full` + 라벨 `text-sm` |
| 선택 | 클릭 즉시 변경 (낙관적) |

### 4.3 담당자 변경 Popover

```
[담당자 변경] 클릭
      │
      └─ Popover 열림 (w-[240px])
         ┌──────────────────────────┐
         │ 🔍 멤버 검색...          │  ← Input
         │ ────────────────────────│
         │ ☑ 👤 Luke Kim           │  ← 다중 선택
         │ ☐ 👤 Kim Park           │
         │ ☐ 👤 Park Choi          │
         │ ────────────────────────│
         │ [적용]                   │  ← Button primary size="sm"
         └──────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `Popover` (shadcn/ui), `w-[240px]` |
| 검색 | `Input` size=`sm`, `Search` 아이콘 접두사 |
| 항목 | `Checkbox` + 아바타 `w-5 h-5` + 이름 `text-sm` |
| 선택 | 다중 선택 후 `[적용]` 버튼으로 확정 |
| 동작 | 선택된 모든 이슈의 담당자를 **설정된 값으로 교체** (추가가 아닌 교체) |

### 4.4 라벨 변경 Popover

```
[라벨 변경] 클릭
      │
      └─ Popover 열림 (w-[240px])
         ┌──────────────────────────┐
         │ 🔍 라벨 검색...          │  ← Input
         │ ────────────────────────│
         │ ☑ ● frontend            │  ← 다중 선택
         │ ☐ ● backend             │     라벨 색상 dot + 이름
         │ ☐ ● urgent              │
         │ ────────────────────────│
         │ [추가] [교체]            │  ← 모드 선택
         └──────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `Popover` (shadcn/ui), `w-[240px]` |
| 검색 | `Input` size=`sm`, `Search` 아이콘 접두사 |
| 항목 | `Checkbox` + 색상 dot `w-2.5 h-2.5 rounded-full` + 라벨명 `text-sm` |
| 모드 | 하단 `ButtonGroup`: `"추가"` (기존 라벨에 추가) / `"교체"` (기존 라벨을 교체) |
| 기본 모드 | `"추가"` |

## 5. 벌크 삭제

### 5.1 삭제 확인 Dialog

```
┌─────────────────────────────────────────────┐
│                                             │
│  이슈 삭제                                   │  ← text-lg font-semibold
│                                             │
│  선택한 3건의 이슈를 삭제하시겠습니까?          │  ← text-sm text-muted-foreground
│  이 작업은 되돌릴 수 없습니다.                 │
│                                             │
│                    [취소]  [삭제]             │  ← 취소: outline, 삭제: destructive
│                                             │
└─────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `Dialog` (shadcn/ui) |
| 너비 | `max-w-[400px]` (소형 모달) |
| 제목 | `"이슈 삭제"`, `text-lg font-semibold` |
| 설명 | `"선택한 {N}건의 이슈를 삭제하시겠습니까?"` + `"이 작업은 되돌릴 수 없습니다."` |
| 취소 | `Button` variant=`outline` |
| 삭제 | `Button` variant=`destructive` |
| 키보드 | `Enter`=삭제 실행, `Esc`=취소 |

## 6. 피드백 & 로딩

### 6.1 벌크 액션 로딩 상태

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  3건 선택됨     [상태 변경 ▼] [⏳ 처리 중...] [담당자 변경] [라벨 변경]       │
│                                disabled     disabled    disabled           │
│                                                        [전체 해제] [삭제]  │
│                                                         disabled  disabled │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

| 상태 | 동작 |
|------|------|
| 변경 진행 중 | 실행한 버튼에 `Loader2` 스피너 + `"처리 중..."` 텍스트, 나머지 버튼 `disabled` |
| 변경 성공 | 선택 해제, 토스트: `"{N}건 업데이트 완료"`, 선택 바 닫힘 |
| 변경 실패 | 토스트: `"업데이트에 실패했습니다. 다시 시도해주세요."` (error), 선택 유지 |
| 삭제 성공 | 선택 해제, 토스트: `"{N}건 삭제 완료"`, 선택 바 닫힘 |
| 삭제 실패 | 토스트: `"삭제에 실패했습니다. 다시 시도해주세요."` (error), 선택 유지 |

## 7. 키보드 단축키

| 키 | 동작 | 조건 |
|------|------|------|
| `X` | 포커스된 행의 선택 토글 | 리스트 뷰, 포커스된 행 있을 때 |
| `Esc` | 모든 선택 해제 (Side Panel 열려있으면 먼저 닫기) | 1건 이상 선택 상태 |
| `Shift + J` | 포커스 이동 + 범위 선택 확장 | 선택 모드 |
| `Shift + K` | 포커스 이동 + 범위 선택 확장 | 선택 모드 |

## 8. 접근성 (Accessibility)

### 8.1 ARIA

| 요소 | ARIA |
|------|------|
| 선택 바 | `role="toolbar" aria-label="선택된 이슈 액션"` |
| 선택 카운트 | `aria-live="polite"` (변경 시 자동 알림) |
| 헤더 체크박스 | `aria-label="전체 선택"`, `indeterminate` 상태 지원 |
| 행 체크박스 | `aria-label="{이슈키} 선택"` |
| 액션 버튼 | 각각 `aria-label` 적용 (예: `"선택된 이슈 상태 변경"`) |
| 삭제 Dialog | `role="alertdialog" aria-labelledby="제목" aria-describedby="설명"` |

### 8.2 스크린 리더

- 선택 변경 시: `"{N}건 선택됨"` 알림 (`aria-live="polite"`)
- 벌크 변경 성공 시: `"{N}건 업데이트 완료"` 알림
- 벌크 삭제 성공 시: `"{N}건 삭제 완료"` 알림
