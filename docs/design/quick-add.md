# Worknest Quick Add 컴포넌트 설계 (CP2-DS-3)

> 이 문서는 이슈를 빠르게 생성하기 위한 Quick Add 컴포넌트의 레이아웃, 인터랙션, 낙관적 업데이트 동작을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

Quick Add는 이슈 생성의 마찰을 최소화하는 인라인 컴포넌트입니다. 별도의 모달을 열지 않고, 리스트/보드 뷰에서 즉시 이슈를 생성할 수 있습니다.

### 1.1 핵심 원칙

- **즉시성**: 트리거 → 입력 → Enter → 생성, 최소 스텝
- **연속성**: 한 이슈 생성 후 다음 입력으로 자동 포커스 유지
- **컨텍스트**: 현재 뷰의 상태(보드 컬럼 등)를 기본값으로 자동 적용
- **낙관적**: 생성 즉시 UI에 반영, 서버 응답 후 확정

## 2. 트리거

### 2.1 키보드 트리거

| 트리거 | 동작 | 조건 |
|--------|------|------|
| `C` 키 | Quick Add 활성화 | 이슈 뷰 컨텍스트, input/textarea/editor 미포커스 |

### 2.2 버튼 트리거

| 위치 | 버튼 | 동작 |
|------|------|------|
| Page Header | `[＋ 이슈]` primary 버튼 | 리스트 맨 위에 Quick Add 활성화 |
| 리스트 뷰 | `[＋ 이슈 추가]` ghost 버튼 (리스트 하단) | 해당 위치에 Quick Add 활성화 |
| 보드 뷰 | `[＋]` ghost 버튼 (컬럼 하단) | 해당 컬럼에 Quick Add 활성화 |

## 6. 인터랙션 흐름

### 6.1 생성 흐름

```
[트리거] → [Quick Add 활성화 + 자동 포커스]
                    │
                    ├─ 제목 입력
                    │
                    ├─ [Enter] → 이슈 생성 (낙관적 업데이트)
                    │             Quick Add 유지 (연속 생성)
                    │             이전 입력 위/아래에 새 행 추가
                    │
                    ├─ [Esc] → Quick Add 비활성화
                    │           입력 내용 삭제
                    │
                    └─ [포커스 아웃] → 입력 내용 있으면 생성
                                      입력 내용 없으면 비활성화
```

### 6.2 키보드 동작

| 키 | 동작 |
|------|------|
| `Enter` | 현재 제목으로 이슈 생성 + Quick Add 유지 (연속 생성 모드) |
| `Shift+Enter` | 현재 제목으로 이슈 생성 + Quick Add 비활성화 |
| `Esc` | Quick Add 비활성화, 입력 내용 삭제 |
| `Tab` | 기본 동작 무시 (Quick Add 내에서는 제목 입력만 지원) |

### 6.3 연속 생성

```
[C] → Quick Add 활성화
        │
        ├─ "로그인 페이지 구현" 입력 → [Enter]
        │   → WRK-4 생성 (낙관적)
        │   → Quick Add 포커스 유지 (빈 입력)
        │
        ├─ "회원가입 API 구현" 입력 → [Enter]
        │   → WRK-5 생성 (낙관적)
        │   → Quick Add 포커스 유지 (빈 입력)
        │
        └─ [Esc] → Quick Add 비활성화
```

## 7. 낙관적 업데이트 (Optimistic Update)

### 7.1 흐름

```
[Enter] → 임시 ID 부여 (temp-{uuid})
        → UI에 즉시 이슈 행/카드 추가
        → API 호출 (POST /api/issues)
        │
        ├─ 성공: 임시 ID → 실제 ID 교체
        │         이슈 키 표시 (WRK-42)
        │
        └─ 실패: 임시 행 제거
                 토스트: "이슈 생성에 실패했습니다. 다시 시도해주세요."
```

### 7.2 임시 이슈 표시

```
리스트 뷰:
┌──────────────────────────────────────────────────────────┐
│  ☐  ···    로그인 페이지 구현         ● Backlog   None   │  ← 임시 상태
│            text-foreground/70                            │     이슈 키 = 로딩 dot (···)
│                                                          │     opacity: 0.7
└──────────────────────────────────────────────────────────┘

서버 응답 후:
┌──────────────────────────────────────────────────────────┐
│  ☐  WRK-4  로그인 페이지 구현         ● Backlog   None   │  ← 확정 상태
│            text-foreground                               │     이슈 키 표시
│                                                          │     opacity: 1
└──────────────────────────────────────────────────────────┘
```

| 속성 | 임시 상태 | 확정 상태 |
|------|----------|----------|
| 이슈 키 | `···` (로딩 dot 3개, `text-muted-foreground`) | `WRK-42` |
| 텍스트 opacity | `0.7` | `1.0` |
| 클릭 | 비활성 (상세 열기 불가) | 활성 |
| 전환 | 서버 응답 시 fade 150ms | — |

### 7.3 보드 뷰 임시 카드

```
┌───────────────────────────┐
│                           │
│  ···                      │  ← 로딩 dot, text-muted-foreground
│  로그인 페이지 구현         │  ← opacity: 0.7
│                           │
└───────────────────────────┘
```

## 10. 접근성 (Accessibility)

### 10.1 포커스 관리

- Quick Add 활성화 시 입력 필드에 자동 포커스
- `Esc` 시 트리거 요소로 포커스 복귀
- 연속 생성 시 포커스는 Quick Add 입력에 유지

### 10.2 ARIA

- Quick Add 행: `role="form" aria-label="이슈 빠른 생성"`
- 입력 필드: `aria-label="이슈 제목"`, `aria-placeholder="이슈 제목을 입력하세요..."`
- 임시 이슈 행: `aria-busy="true" aria-label="생성 중..."`
- `[＋ 이슈]` 버튼: `aria-label="이슈 추가"`
- `[＋ 이슈 추가]` 버튼: `aria-label="이슈 추가"`

### 10.3 스크린 리더

- 이슈 생성 성공 시: `aria-live="polite"` 영역에 `"WRK-42 이슈가 생성되었습니다"` 알림
- 이슈 생성 실패 시: `aria-live="assertive"` 영역에 `"이슈 생성에 실패했습니다"` 알림
