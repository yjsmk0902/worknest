# Worknest 앱 셸 레이아웃 설계

> 이 문서는 로그인 후 표시되는 메인 앱 셸의 레이아웃, 사이드바, 콘텐츠 영역, Side Panel, 반응형 동작을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

### 2.2 사이드바 컴포넌트 스펙

#### Org/WS 선택기

```
클릭 시:
┌──────────────────────────────────────┐
│ ┌──────────────────────────────────┐ │
│ │ 🔍 조직/워크스페이스 검색...       │ │  ← 입력 필드
│ └──────────────────────────────────┘ │
│                                      │
│  Acme Corporation                    │  ← 조직명 (text-xs uppercase
│  ┌──────────────────────────────┐    │     text-muted-foreground)
│  │ ✓ 개발팀                      │    │  ← 현재 선택된 WS
│  │   디자인팀                     │    │     ✓ = CheckIcon
│  │   마케팅팀                     │    │
│  └──────────────────────────────┘    │
│                                      │
│  Monolith Projects                   │  ← 다른 조직
│  ┌──────────────────────────────┐    │
│  │   Core Team                   │    │
│  └──────────────────────────────┘    │
│                                      │
│  ─────────────────────────────────── │
│  ＋ 새 워크스페이스 만들기             │
│  ⚙ 조직 설정                         │
│                                      │
└──────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 트리거 | 사이드바 상단, `h-12`, `hover:bg-accent`, `cursor-pointer` |
| 드롭다운 | `w-[280px]`, Radix `DropdownMenu` 또는 `Popover` |
| 검색 | 즉시 필터 (클라이언트 사이드) |
| 키보드 | `↑↓` 이동, `Enter` 선택, `Esc` 닫기 |
| 조직 구분 | 조직명을 그룹 헤더로 |
| 현재 선택 | `Check` 아이콘 (Lucide, 16px) |

#### My Work 섹션

| 항목 | 아이콘 | 뱃지 | 라우트 |
|------|--------|------|--------|
| Inbox | `Bell` | 미읽은 알림 수 | `/:org/:ws/my/inbox` |
| My Issues | `CircleUser` | 할당된 이슈 수 | `/:org/:ws/my/issues` |
| Favorites | `Star` | 없음 | `/:org/:ws/my/favorites` |

뱃지 스타일: `bg-primary text-primary-foreground text-xs rounded-full min-w-[20px] h-5 px-1.5 inline-flex items-center justify-center`

#### Projects 섹션

| 요소 | 설명 |
|------|------|
| 프로젝트 항목 | `[아이콘] [프로젝트명]` — 클릭 시 이슈 리스트로 이동 |
| 확장 화살표 | `ChevronRight` → 클릭 시 하위 메뉴 확장 (90도 회전) |
| 추가 버튼 | `＋ 프로젝트 추가` — 프로젝트 생성 모달 |
| 즐겨찾기 | 드래그 또는 우클릭 컨텍스트 메뉴로 추가 |

프로젝트 확장 시:

```
│  📁  WORK         ▼   │
│     📋  Issues        │  ← indent 24px
│     📊  Board         │
│     🔄  Cycles        │
│     ⚙  Settings      │
```

#### Wiki 섹션

| 요소 | 설명 |
|------|------|
| 스페이스 항목 | `[아이콘] [스페이스명]` — 클릭 시 스페이스 페이지 목록 |
| 확장 화살표 | `ChevronRight` — 페이지 트리 표시 |
| 추가 버튼 | `＋ 스페이스 추가` — 스페이스 생성 모달 |

스페이스 확장 시 (페이지 트리):

```
│  📄  DevOps        ▼   │
│     📄 배포 가이드       │  ← indent 24px (레벨 0)
│       📄 AWS 설정       │  ← indent 40px (레벨 1)
│         📄 EC2 설정     │  ← indent 52px (레벨 2, 최대)
│     📄 모니터링         │
│     ＋ 페이지 추가       │
```

#### 사용자 영역 (하단)

```
┌──────────────────────┐
│  [아바타 32px] Luke   │  ← 클릭 시 팝오버 메뉴
│              Kim      │
└──────────────────────┘

팝오버 메뉴:
┌──────────────────────┐
│  프로필               │
│  테마                 │  ← 서브메뉴: 라이트/다크/시스템
│  ───────────────────  │
│  워크스페이스 설정      │
│  조직 설정             │
│  ───────────────────  │
│  단축키 (Cmd+/)       │
│  ───────────────────  │
│  로그아웃              │  ← text-destructive
└──────────────────────┘
```

### 2.3 축소 상태 (48px)

```
┌────┐
│    │
│ [W]│  ← Org/WS 이니셜 (클릭 시 선택기 팝오버)

### 2.4 사이드바 토글

|------|------|

| 애니메이션 | 너비 변화, `transition: width 150ms ease` |

## 3. 메인 콘텐츠 영역

### 3.1 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  ┌─ Page Header ────────────────────────────────────┐   │
│  │                                                   │   │
│  │  [Breadcrumb 또는 페이지 제목]        [액션 버튼들]  │   │
│  │                                                   │   │
│  └───────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─ Page Content ───────────────────────────────────┐   │
│  │                                                   │   │
│  │  (라우트별 콘텐츠)                                  │   │
│  │                                                   │   │
│  │  max-width: 1200px                                │   │
│  │  padding: px-6 py-6                               │   │
│  │                                                   │   │
│  └───────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 Page Header 컴포넌트

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  [아이콘] 페이지 제목                [필터] [뷰] [＋]     │
│  (부제목 또는 Breadcrumb)                                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 높이 | 자동 (내용에 따라) |
| 패딩 | `px-6 py-4` |
| 배경 | `bg-background` |
| 하단 보더 | `border-b border-border` |
| 제목 | `text-xl font-semibold` 또는 `text-lg font-semibold` |
| 부제목/Breadcrumb | `text-sm text-muted-foreground` |
| 액션 버튼 | 우측 정렬, `flex gap-2` |

#### 화면별 Page Header

| 화면 | 좌측 | 우측 |
|------|------|------|
| My Issues | `📋 My Issues` | — |
| Inbox | `🔔 Inbox` | 모두 읽음 |
| 프로젝트 이슈 (리스트) | `[프로젝트 아이콘] WORK Issues` | `[필터] [정렬] [뷰 전환] [＋ 이슈]` |
| 프로젝트 이슈 (보드) | `[프로젝트 아이콘] WORK Board` | `[필터] [＋ 이슈]` |
| 사이클 | `🔄 Cycles` | `[＋ 사이클]` |
| Wiki 스페이스 | `📄 [스페이스명]` | `[＋ 페이지]` |
| Wiki 페이지 | Breadcrumb: `스페이스 > 부모 > 현재` | `[저장 상태] [공유]` |
| 설정 | `⚙ Settings` | — |

### 3.3 콘텐츠 스크롤

```
┌── Sidebar ──┬── Main ────────────────────────────────┐
│             │  ┌─ Header ─────────────────────────┐  │  ← 고정 (sticky)
│  (고정,      │  │ 제목              [액션]          │  │
│   스크롤      │  └────────────────────────────────────┘  │
│   독립)      │  ┌─ Content ───────────────────────┐  │  ← 스크롤
│             │  │                                   │  │     overflow-y: auto
│             │  │                                   │  │
│             │  │                                   │  │
│             │  └───────────────────────────────────┘  │
└─────────────┴────────────────────────────────────────┘
```

- 사이드바: 자체 스크롤 (`overflow-y: auto`, 스크롤바 숨김)
- 메인 헤더: `sticky top-0 z-10 bg-background`
- 메인 콘텐츠: `overflow-y: auto`, 전체 높이 사용

### 4.4 1280px+ (인라인)

```
┌── Sidebar ──┬── Main (축소) ──────────┬── Side Panel ──────┐

### 4.5 1024~1279px (오버레이)

```
┌── 48px ──┬── Main ──────────────────────┐
│          │                              │
│ Sidebar  │  (원래 너비 유지)              │

│          │                    ┌─────────┤
│          │                    │ Side    ││
│          │    (반투명 백드롭)   │ Panel   ││  ← 오버레이
│          │                    │ 640px   ││     z-40
│          │                    │         ││     backdrop: bg-black/50
│          │                    └─────────┤│
│          │                              │
└──────────┴──────────────────────────────┘
```

- Side Panel: `position: fixed`, `right: 0`, `z-40`
- 백드롭: `bg-black/50`, 클릭 시 Panel 닫힘
- 메인 콘텐츠: 원래 너비 유지 (축소 안 됨)

### 4.6 Side Panel 상태

| 상태 | 설명 |
|------|------|
| 닫힘 | Side Panel 미표시 |
| 열림 | 이슈 상세 로드됨 |
| 로딩 | Panel 열린 상태, 본문 영역에 스켈레톤 |
| 다른 이슈 전환 | 리스트에서 다른 이슈 클릭 시, Panel 내용만 교체 (재열기 없음) |
| 에러 | `"이슈를 불러올 수 없습니다"` + 재시도 버튼 |

## 6. 내비게이션 패턴

### 6.1 글로벌 단축키

| 단축키 | 동작 | 조건 |
|--------|------|------|
| `Cmd+\` | 사이드바 토글 (열림/축소) | 항상 |
| `Cmd+K` | 커맨드 팔레트 열기 | 항상 |
| `Cmd+/` | 단축키 도움말 시트 | 항상 |

### 6.2 라우트 전환

```
사이드바 클릭 → 즉시 라우트 전환 (트랜지션 없음)
뒤로/앞으로 → 브라우저 히스토리 (pushState)
Cmd+K 검색 결과 선택 → 해당 라우트로 이동
```

### 6.3 활성 경로 표시

사이드바 항목은 현재 라우트와 매칭 시 활성 상태로 표시합니다.

```
비활성:  text-muted-foreground
활성:    bg-accent text-accent-foreground font-medium rounded-md
호버:    bg-accent/50 rounded-md
```

매칭 규칙:
- My Work 항목: 정확한 경로 매치 (`/:org/:ws/my/inbox`)
- 프로젝트: 프로젝트 경로 접두사 매치 (`/:org/:ws/projects/:prefix/*`)
- Wiki: 스페이스 경로 접두사 매치 (`/:org/:ws/wiki/:spaceSlug/*`)

### 6.4 Breadcrumb

Wiki 페이지에서 사용:

```
스페이스명  /  상위 페이지  /  현재 페이지
  (링크)       (링크)        (현재, 굵게)
```

| 속성 | 값 |
|------|-----|
| 구분자 | `/` 또는 `ChevronRight` (Lucide, 12px) |
| 텍스트 | `text-sm text-muted-foreground` |
| 현재 | `text-sm text-foreground font-medium` |
| 호버 | 밑줄 |
| 영역 | `px-6 py-2`, 헤더 상단 |

## 8. 토스트 알림 (Sonner)

### 8.1 위치 및 스타일

```
                                    ┌──────────────────────────┐
                                    │ ✓ 이슈가 생성되었습니다    │
                                    │                          │
                                    │              [실행 취소]  │
                                    └──────────────────────────┘
                                                  ↑
                                         우하단, gap-2 스택
```

| 속성 | 값 |
|------|-----|
| 위치 | 우하단 (`bottom-right`) |
| 너비 | **356px** |
| 자동 닫힘 | **3초** |
| 최대 표시 | 3개 스택 |
| z-index | `100` |

### 8.2 토스트 유형

| 유형 | 아이콘 | 색상 | 예시 |
|------|--------|------|------|
| 성공 | `CheckCircle` | `text-green-500` | "이슈가 생성되었습니다" |
| 에러 | `XCircle` | `text-destructive` | "저장에 실패했습니다" |
| 정보 | `Info` | `text-blue-500` | "새 알림이 있습니다" |
| 경고 | `AlertTriangle` | `text-yellow-500` | "마감일이 지났습니다" |

### 8.3 토스트 + 실행 취소 (Undo)

낙관적 업데이트 작업(상태 변경, 삭제 등)에서:

```
┌──────────────────────────────────────────┐
│ ✓ 이슈 상태가 'Done'으로 변경되었습니다     │
│                                          │
│                               [실행 취소] │  ← ghost 버튼
└──────────────────────────────────────────┘
```

- "실행 취소" 클릭 시: 이전 상태로 복원, 서버에 롤백 요청
- 타임아웃(3초) 후: 실행 취소 불가, 서버 확정
