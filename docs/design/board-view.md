# Worknest 보드 (칸반) 뷰 설계 (CP3-DS-2)

> 이 문서는 칸반 보드 뷰의 컬럼 레이아웃, 카드 디자인, 드래그 앤 드롭 피드백, Quick Add, 빈 상태를 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

보드 뷰는 이슈를 상태 카테고리별 컬럼으로 그룹화하여 칸반 방식으로 표시합니다. 드래그 앤 드롭으로 이슈 상태를 직관적으로 변경할 수 있습니다.

### 1.1 핵심 원칙

- **시각적 흐름**: 왼쪽(Backlog) → 오른쪽(Completed)으로 이슈 진행 현황을 한눈에 파악
- **직관적 DnD**: 카드를 끌어서 컬럼 간 이동 = 상태 변경 (낙관적 업데이트)
- **정보 요약**: 카드에 핵심 속성만 표시 (키, 제목, 우선순위, 담당자, 라벨, 마감일)
- **연속 생성**: 각 컬럼 하단 Quick Add로 해당 상태의 이슈를 즉시 생성

```

│  ┌─ Backlog ──┐  ┌─ Todo ─────┐  ┌─ In Prog ──┐  ┌─ Done ─────┐  ┌─ Cancel─┐
│  │            │  │            │  │            │  │            │  │          │
│  │  [카드]    │  │  [카드]    │  │  [카드]    │  │            │  │          │
│  │  [카드]    │  │  [카드]    │  │            │  │            │  │          │
│  │  [카드]    │  │            │  │            │  │            │  │          │
│  │            │  │            │  │            │  │            │  │          │
│  │  ─ ─ ─ ─  │  │  ─ ─ ─ ─  │  │  ─ ─ ─ ─  │  │  ─ ─ ─ ─  │  │ ─ ─ ─ ─ │
│  │  ＋        │  │  ＋        │  │  ＋        │  │  ＋        │  │ ＋       │
│  └────────────┘  └────────────┘  └────────────┘  └────────────┘  └──────────┘
│  ← ─ ─ ─ ─ ─ ─ 수평 스크롤 (overflow-x-auto) ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ → │

└──────────────────────────────────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 레이아웃 | `flex` (수평), `overflow-x-auto` |
| 간격 | `gap-3` (12px) |
| 패딩 | `px-4 pb-4` |
| 높이 | `calc(100vh - 헤더 - 툴바)`, 세로 전체 영역 |

## 3. 컬럼 디자인

### 3.1 컬럼 구조

```
┌─ 컬럼 ───────────────────────────────────┐
│                                           │
│  ┌─ 헤더 ──────────────────────────────┐  │
│  │ ● Todo                          (5) │  │  ← h-10, px-3
│  │ 색상dot  상태명              카운트   │  │
│  └─────────────────────────────────────┘  │
│                                           │
│  ┌─ 카드 영역 (스크롤) ───────────────┐   │
│  │                                    │   │
│  │  [카드 1]                          │   │
│  │  [카드 2]                          │   │
│  │  [카드 3]                          │   │
│  │                                    │   │
│  └────────────────────────────────────┘   │
│                                           │
│  ┌─ Quick Add ─────────────────────────┐  │
│  │  ＋                                 │  │  ← border-t, mt-auto
│  └─────────────────────────────────────┘  │
│                                           │
└───────────────────────────────────────────┘
```

### 3.2 컬럼 스타일

| 속성 | 값 |
|------|-----|
| 최소 너비 | `min-w-[280px]` |
| 최대 너비 | `max-w-[320px]` |
| 배경 | `bg-muted/30` |
| 모서리 | `rounded-lg` |
| 레이아웃 | `flex flex-col` |
| 높이 | 부모 높이 전체 (`h-full`) |

### 3.3 컬럼 헤더

| 속성 | 값 |
|------|-----|
| 높이 | `h-10` (40px) |
| 패딩 | `px-3` |
| 레이아웃 | `flex items-center gap-2` |
| 상태 색상 dot | `w-2.5 h-2.5 rounded-full`, 색상: `--status-{category}-text` |
| 상태명 | `text-sm font-medium text-foreground` |
| 카운트 뱃지 | `text-xs text-muted-foreground ml-auto` |

### 3.4 카드 영역

| 속성 | 값 |
|------|-----|
| 레이아웃 | `flex-1 overflow-y-auto` |
| 패딩 | `px-2 py-1` |
| 간격 | 카드 사이 `mb-2` (8px) |

### 3.5 컬럼 Quick Add

| 속성 | 값 |
|------|-----|
| 위치 | `mt-auto` (컬럼 하단 고정) |
| 보더 | `border-t border-border` |
| 패딩 | `p-2` |
| 버튼 | `Plus` 아이콘 (Lucide, 16px), `text-muted-foreground hover:text-foreground` |
| 동작 | 클릭 시 Quick Add 카드 활성화 (`quick-add.md` 참조) |

## 4. 카드 디자인

### 4.1 카드 레이아웃

```
┌───────────────────────────────────────┐
│                                       │
│  WRK-42                               │  ← 이슈 키
│  로그인 시 비밀번호 에러                  │  ← 제목 (최대 2줄)
│  메시지 미표시                           │
│                                       │
│  🔴  👤👤  ● ●  04/15                  │  ← 하단 메타 정보
│                                       │
└───────────────────────────────────────┘
```

### 4.2 카드 스타일

| 속성 | 값 |
|------|-----|
| 배경 | `bg-card` |
| 보더 | `border border-border` |
| 모서리 | `rounded-md` |
| 패딩 | `p-3` |
| 하단 간격 | `mb-2` |
| 커서 | `cursor-grab` (호버 시), `cursor-grabbing` (드래그 중) |
| 호버 | `shadow-sm border-border/80` (미세한 그림자 추가) |

### 4.3 카드 내부 요소

#### 이슈 키

| 속성 | 값 |
|------|-----|
| 텍스트 | `text-xs font-mono text-muted-foreground` |
| 예시 | `WRK-42` |

#### 제목

| 속성 | 값 |
|------|-----|
| 텍스트 | `text-sm font-medium` |
| 줄 수 | `line-clamp-2` (최대 2줄, 초과 시 말줄임표) |
| 상단 간격 | `mt-1` |

#### 하단 메타 행

| 속성 | 값 |
|------|-----|
| 레이아웃 | `flex items-center gap-2 mt-2` |
| 조건 | 우선순위, 담당자, 라벨, 마감일 중 하나라도 있을 때만 표시 |

### 4.4 하단 메타 요소 상세

#### 우선순위 아이콘

| 속성 | 값 |
|------|-----|
| 크기 | `w-4 h-4` |
| 색상 | `PRIORITY_CONFIG` 색상 매핑 (urgent=`--priority-urgent`, high=`--priority-high`, etc.) |
| None 우선순위 | 미표시 (카드에서 생략) |

#### 담당자 아바타

| 속성 | 값 |
|------|-----|
| 크기 | `w-5 h-5 rounded-full` |
| 최대 표시 | 2명 |
| 겹침 | 두 번째 아바타에 `-ml-1` |
| 3명 이상 | 2명 표시 + `+N` 뱃지 (`text-xs text-muted-foreground w-5 h-5 bg-muted rounded-full flex items-center justify-center`) |
| 위치 | `ml-auto` (우측 정렬) |

#### 라벨 색상 dot

| 속성 | 값 |
|------|-----|
| 크기 | `w-2 h-2 rounded-full` |
| 최대 표시 | 3개 |
| 색상 | 각 라벨의 `color` 값 |
| 4개 이상 | 3개 표시 (초과분 미표시) |

#### 마감일 (조건부 표시)

| 조건 | 스타일 |
|------|--------|
| 초과 (overdue) | `text-xs text-destructive font-medium` |
| 3일 이내 임박 | `text-xs text-orange-500` |
| 3일 초과 | 미표시 (카드에서 생략) |

> 마감일은 임박하거나 초과한 경우에만 카드에 표시하여 정보 밀도를 최적화합니다.

## 5. 드래그 앤 드롭

### 5.1 DnD 피드백 토큰

#### 드래그 중인 카드 (Dragging Card)

```
┌───────────────────────────────────────┐
│                                       │  ← opacity-85
│  WRK-42                               │     shadow-lg
│  로그인 시 비밀번호 에러                  │     scale-[1.02]
│  메시지 미표시                           │     cursor-grabbing
│                                       │     rotate-[1deg] (미세 회전)
│  🔴  👤👤  04/15                       │
│                                       │
└───────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 투명도 | `opacity-85` |
| 그림자 | `shadow-lg` |
| 스케일 | `scale-[1.02]` |
| 커서 | `cursor-grabbing` |
| 회전 | `rotate-[1deg]` (미세 회전으로 들어올린 느낌) |

#### 원위치 플레이스홀더 (Placeholder)

```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
│                                       │  ← border-2 border-dashed border-border
│                                       │     bg-muted/30
│                                       │     같은 높이 유지
│                                       │
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

| 속성 | 값 |
|------|-----|
| 보더 | `border-2 border-dashed border-border` |
| 배경 | `bg-muted/30` |
| 높이 | 드래그 시작 카드와 동일한 높이 유지 |
| 모서리 | `rounded-md` |

#### 드롭 존 표시 (Drop Zone Indicator)

```
┌─ Todo ─────────────────────┐
│                            │  ← ring-2 ring-primary/30
│  [카드]                    │     bg-primary/5 (컬럼 전체)
│  [카드]                    │
│                            │
│  ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐│  ← 삽입 위치 표시
│  │ 여기에 놓기            ││     border-2 border-dashed border-primary/30
│  └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘│     h-10
│                            │
│  [카드]                    │
│                            │
└────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 타겟 컬럼 링 | `ring-2 ring-primary/30` |
| 컬럼 배경 | `bg-primary/5` |
| 삽입 위치 | `border-2 border-dashed border-primary/30 rounded-md h-10` |

#### 카드 리플로우 애니메이션

| 속성 | 값 |
|------|-----|
| 전환 | `transition-transform duration-150 ease-out` |
| 적용 대상 | 드래그 중 위치가 변경되는 나머지 카드들 |

### 5.2 자동 스크롤 (Auto-scroll)

| 속성 | 값 |
|------|-----|
| 감지 영역 | 보드 컨테이너의 좌/우 **60px** 이내 |
| 스크롤 속도 | 가장자리에 가까울수록 빠르게 (선형 보간) |
| 방향 | 수평 스크롤 (`scrollLeft` 조정) |

### 5.3 드롭 동작

| 동작 | 설명 |
|------|------|
| 같은 컬럼 내 | 순서 변경 (sort_order 업데이트, fractional indexing) |
| 다른 컬럼으로 | 상태 변경 (낙관적) + 순서 변경 |
| 낙관적 업데이트 | 드롭 즉시 UI 반영, API 호출, 실패 시 롤백 + 토스트 에러 |

```
│                              │

```

| 속성 | 값 |
|------|-----|
| 텍스트 | `"이슈 없음"` |
| 스타일 | `text-sm text-muted-foreground` |
| 정렬 | `flex-1 flex items-center justify-center` |

### 6.2 빈 보드 (Empty Board — 모든 컬럼 비어있음)

```
┌─────────────────────────────────────────────────────┐
│                                                      │
│                                                      │
│               [Columns 아이콘, 48px]                  │
│               text-muted-foreground                  │
│                                                      │
│        이슈를 만들어 보드를 시작하세요                    │
│        text-lg font-medium                           │
│                                                      │
│        카드를 드래그하여 상태를 변경할 수 있습니다         │
│        text-sm text-muted-foreground                 │
│                                                      │
│        [＋ 이슈 만들기]  Button primary                │
│                                                      │
│                                                      │
└─────────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 표시 조건 | 모든 컬럼의 이슈 수가 0일 때 |
| 레이아웃 | 보드 영역 중앙, `flex flex-col items-center justify-center gap-3` |
| 아이콘 | `Columns` (Lucide, 48px), `text-muted-foreground` |
| 제목 | `text-lg font-medium` |
| 설명 | `text-sm text-muted-foreground` |
| CTA | `Button` variant=`primary`, `Plus` 아이콘 + `"이슈 만들기"` |

## 7. 카드 클릭 동작

| 동작 | 결과 |
|------|------|
| 클릭 | Side Panel로 이슈 상세 열기 (1024px+ 화면) |
| 더블클릭 / Enter | Full Page로 이슈 상세 열기 |
| 드래그 | DnD 시작 (클릭과 구분: 150ms 유지 또는 5px 이동 시 드래그 판정) |

## 9. 접근성 (Accessibility)

### 9.1 ARIA

| 요소 | ARIA |
|------|------|
| 보드 컨테이너 | `role="region" aria-label="칸반 보드"` |
| 컬럼 | `role="list" aria-label="{상태명} 컬럼, {N}개 이슈"` |
| 카드 | `role="listitem" aria-grabbed={isDragging} aria-label="{이슈키} {제목}"` |
| 드래그 핸들 | `aria-roledescription="드래그 가능한 이슈 카드"` |

### 9.2 키보드 DnD (대체 방법)

드래그 앤 드롭을 사용할 수 없는 환경에서는 키보드로 이슈 상태를 변경할 수 있습니다:

1. 카드에 포커스 → `Enter`로 상세 열기
2. 상세에서 상태 드롭다운 변경

### 9.3 스크린 리더

- 드래그 시작: `"{이슈키}를 잡았습니다. 화살표 키로 이동하세요"` 알림
- 드롭 완료: `"{이슈키}를 {상태명} 컬럼으로 이동했습니다"` 알림
- 드롭 취소: `"이동이 취소되었습니다"` 알림
