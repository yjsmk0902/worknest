# Worknest 커맨드 팔레트 설계 (CP6-DS-1)

> 이 문서는 글로벌 검색 및 명령어 실행을 위한 커맨드 팔레트(Cmd+K)의 레이아웃, 검색 동작, 명령어 모드, 키보드 내비게이션을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

커맨드 팔레트는 이슈, Wiki 페이지, 프로젝트를 통합 검색하고 빠른 명령어를 실행하는 키보드 중심 UI입니다. `Cmd+K` (macOS) / `Ctrl+K` (Windows/Linux) 로 어디서든 열 수 있습니다.

### 1.1 핵심 원칙

- **즉시 접근**: `Cmd+K`는 모든 컨텍스트에서 항상 활성화 (모달 내부 제외)
- **속도**: 디바운스 300ms, 이슈 ID 패턴 감지 시 서버 호출 없이 즉시 이동
- **명령어 모드**: `>` 접두사로 명령어 진입, 검색과 명령 실행을 한 곳에서 처리
- **학습 곡선 최소화**: 최근 항목으로 반복 접근 패턴 지원

## 3. 검색 입력 영역

### 3.1 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  🔍  검색하거나 명령어를 입력하세요...              [⏳] │

└─────────────────────────────────────────────────────────┘
```

### 3.2 입력 스타일

| 속성 | 값 |
|------|-----|
| 높이 | `h-12` (48px) |
| 패딩 | `px-4` |
| 레이아웃 | `flex items-center gap-3` |
| 보더 | `border-b border-border` |
| 배경 | `bg-popover` (컨테이너와 동일) |
| 모서리 | 상단만 `rounded-t-xl` |

### 3.3 검색 아이콘

| 속성 | 값 |
|------|-----|
| 아이콘 | `Search` (Lucide) |
| 크기 | `w-5 h-5` |
| 색상 | `text-muted-foreground` |
| 위치 | 입력 좌측 |

### 3.4 입력 필드

| 속성 | 값 |
|------|-----|
| 포커스 | `autoFocus` — 팔레트 열릴 때 자동 포커스 |
| placeholder | `"검색하거나 명령어를 입력하세요..."` |
| 텍스트 | `text-base font-normal text-foreground` |
| placeholder 색상 | `text-muted-foreground` |
| 배경 | `bg-transparent` |
| 아웃라인 | `outline-none` (팔레트 자체가 포커스 컨테이너) |
| 크기 | `flex-1` |
| 디바운스 | 300ms (입력 후 300ms 대기 후 검색 호출) |

### 3.5 로딩 인디케이터

| 속성 | 값 |
|------|-----|

| 크기 | `w-4 h-4` |

## 5. 결과 항목

### 5.1 최근 항목 (Recent Items)

입력이 비어있을 때 기본 표시됩니다. localStorage에서 최대 5개 표시합니다.

```
┌─────────────────────────────────────────────────────────┐
│  최근 항목                                                │
│  ┌─────────────────────────────────────────────────────┐│
│  │  📄  API 설계 가이드                       2시간 전  ││  ← Wiki 페이지
│  │  FileText                                           ││
│  ├─────────────────────────────────────────────────────┤│
│  │  ●  WORK-142  로그인 버그 수정              어제     ││  ← 이슈
│  │  status dot   issue key                             ││
│  ├─────────────────────────────────────────────────────┤│
│  │  📁  Worknest Backend                      3일 전   ││  ← 프로젝트
│  │  Folder                                             ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 저장소 | localStorage `cmdK_recent` 키 |
| 최대 저장 | 20개 (FIFO) |
| 최대 표시 | 5개 |
| 저장 데이터 | `{ type, id, title, key?, timestamp }` |

## 6. 이슈 ID 즉시 이동

### 6.1 패턴 감지

입력값이 `[A-Z]{2,5}-\d+` 패턴 (예: `WORK-42`, `API-123`)과 일치하면 서버 API 호출 없이 즉시 이동 옵션을 최상단에 표시합니다.

```
┌─────────────────────────────────────────────────────────┐
│  🔍 WORK-42                                             │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐│
│  │  →  WORK-42로 이동                                  ││  ← 하이라이트
│  │  ArrowRight   text-sm font-medium text-primary      ││
│  └─────────────────────────────────────────────────────┘│
│                                                          │
│  이슈                                                    │
│    ...기타 검색 결과...                                   │
└─────────────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 패턴 | `/^[A-Z]{2,5}-\d+$/i` (대소문자 무시) |
| 아이콘 | `ArrowRight` (Lucide, 16px), `text-primary` |
| 텍스트 | `"{KEY}로 이동"`, `text-sm font-medium text-primary` |
| 동작 | Enter 즉시 `/:orgSlug/:wsSlug/projects/:prefix/issues/:key` 로 라우팅 |
| 우선순위 | 항상 결과 목록 최상단 |

## 7. 명령어 모드

### 7.1 진입 조건

입력값이 `>` 문자로 시작하면 명령어 모드로 전환됩니다. `>` 이후 텍스트로 명령어를 필터링합니다.

### 7.2 명령어 목록

```
┌─────────────────────────────────────────────────────────┐
│  🔍 >이슈                                               │
├─────────────────────────────────────────────────────────┤
│  명령어                                                  │
│  ┌─────────────────────────────────────────────────────┐│
│  │  ＋  이슈 생성                                      ││  ← 필터 매칭
│  ├─────────────────────────────────────────────────────┤│
│  │  👤  내 이슈                                        ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

| 명령어 | 아이콘 | 동작 |
|--------|--------|------|
| 이슈 생성 | `Plus` (16px) | 이슈 생성 모달 열기 |
| 프로젝트 이동 | `Folder` (16px) | 프로젝트 선택 서브메뉴 (추후), MVP는 프로젝트 목록 이동 |
| 내 이슈 | `CircleUser` (16px) | My Issues 페이지로 이동 |
| 설정 | `Settings` (16px) | 워크스페이스 설정 페이지로 이동 |
| 로그아웃 | `LogOut` (16px) | 로그아웃 실행 |

### 7.3 명령어 항목 스타일

| 속성 | 값 |
|------|-----|
| 높이 | `h-10` (40px) |
| 패딩 | `px-4` |
| 레이아웃 | `flex items-center gap-3` |
| 아이콘 | 각 명령어별 아이콘, `w-4 h-4 text-muted-foreground` |
| 텍스트 | `text-sm text-foreground` |
| 호버/포커스 | `bg-accent rounded-md` |

## 9. 키보드 내비게이션

### 9.1 단축키 매핑

| 키 | 동작 | 조건 |
|------|------|------|
| `Cmd+K` / `Ctrl+K` | 팔레트 열기/닫기 (토글) | 항상 활성 (모달 내부 제외) |
| `↓` | 다음 결과 항목으로 포커스 이동 | 팔레트 열림 |
| `↑` | 이전 결과 항목으로 포커스 이동 | 팔레트 열림 |
| `Enter` | 포커스된 항목 선택 (이동/실행) | 포커스된 항목 있을 때 |
| `Esc` | 팔레트 닫기 | 팔레트 열림 |

### 9.2 포커스 동작

```
[팔레트 열림]
  ├─ 입력 필드에 autoFocus
  ├─ ↓ 누름 → 첫 번째 결과 항목에 포커스 (bg-accent)
  ├─ 계속 ↓ → 다음 항목으로 이동 (순환: 마지막 → 첫 번째)
  ├─ ↑ 누름 → 이전 항목 (순환: 첫 번째 → 마지막)
  ├─ 입력 변경 → 포커스 인덱스 리셋 (0번 항목)
  └─ Enter → 포커스된 항목의 동작 실행 + 팔레트 닫기
```

### 9.3 포커스 시각 표시

| 속성 | 값 |
|------|-----|
| 포커스된 항목 배경 | `bg-accent` |
| 전환 | `transition-colors duration-100` |
| 스크롤 | 포커스된 항목이 뷰포트 밖이면 `scrollIntoView({ block: 'nearest' })` |

## 10. 접근성 (Accessibility)

### 10.1 ARIA

| 요소 | ARIA |
|------|------|
| 팔레트 컨테이너 | `role="dialog" aria-modal="true" aria-label="글로벌 검색"` |
| 입력 필드 | `role="combobox" aria-expanded="true" aria-controls="cmd-results" aria-activedescendant="{focused-id}"` |
| 결과 목록 | `role="listbox" id="cmd-results"` |
| 결과 항목 | `role="option" id="cmd-result-{index}" aria-selected="{focused}"` |
| 섹션 헤더 | `role="presentation"` (탐색 건너뜀) |

### 10.2 포커스 트랩

팔레트가 열린 동안 Tab 키로 팔레트 외부 요소에 접근할 수 없습니다. `Esc` 또는 백드롭 클릭으로만 닫을 수 있습니다.

### 10.3 스크린 리더

- 팔레트 열림: `"글로벌 검색 열림"` 알림
- 결과 변경: 포커스 이동 시 `aria-activedescendant` 업데이트로 현재 항목 읽기
- 로딩: `aria-busy="true"` 입력 영역에 적용
- 빈 상태: `aria-live="polite"` 영역에 `"결과가 없습니다"` 알림

## 11. 상태 흐름 요약

```
[Cmd+K 입력]
  │
  ├─ 팔레트 열림 (autoFocus)
  │
  ├─ 입력 비어있음 → "최근 항목" 표시 (localStorage, 최대 5개)
  │
  ├─ 입력 변경 (300ms 디바운스)
  │   ├─ 이슈 ID 패턴 매칭 → "{KEY}로 이동" 최상단 표시
  │   ├─ ">" 시작 → 명령어 모드 (명령어 필터링)
  │   └─ 일반 텍스트 → API 검색 호출
  │       ├─ 로딩 중: 입력 우측 Loader2 스피너
  │       ├─ 결과 있음: 카테고리별 그룹 표시
  │       └─ 결과 없음: "결과가 없습니다" 빈 상태
  │
  ├─ 항목 선택 (Enter/클릭)
  │   ├─ 이동 대상으로 라우팅
  │   ├─ 최근 항목에 추가 (localStorage)
  │   └─ 팔레트 닫기
  │
  └─ Esc / 백드롭 클릭 → 팔레트 닫기
```
