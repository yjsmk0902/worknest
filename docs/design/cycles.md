# Worknest 사이클 화면 설계 (CP4-DS-1)

> 이 문서는 사이클 리스트, 생성/편집 모달, 사이클 상세 페이지, 이슈 추가 팝오버, 이월 모달, 빈 상태를 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

사이클은 스프린트와 동일한 개념으로, 일정 기간 동안 수행할 이슈를 그룹화합니다. 프로젝트당 활성 사이클은 최대 1개이며, 사이클 완료 시 미완료 이슈를 다른 사이클로 이월할 수 있습니다.

### 1.1 핵심 원칙

- **직관적 상태**: Draft → Active → Completed 흐름을 색상과 뱃지로 명확하게 표시
- **빠른 이슈 관리**: 팝오버에서 검색 + 체크박스로 다중 이슈를 한 번에 추가
- **진행률 시각화**: 상태 카테고리별 수평 막대로 진척 현황 한눈에 파악
- **이월 지원**: 사이클 완료 시 미완료 이슈를 다음 사이클로 이동하는 흐름

### 5.4 이슈 리스트

사이클 상세의 이슈 리스트는 `list-view.md`에 정의된 리스트 뷰 테이블을 그대로 재사용합니다.

| 속성 | 값 |
|------|-----|
| 데이터 소스 | 해당 사이클에 속한 이슈만 필터링 |
| 정렬/필터 | 리스트 뷰의 정렬/필터 기능 동일 |
| 인라인 편집 | 리스트 뷰와 동일 (상태, 우선순위, 담당자, 마감일) |
| 키보드 | 리스트 뷰와 동일 (J/K 이동, Enter 열기 등) |
| 컨텍스트 메뉴 | 우클릭 시 `"사이클에서 제거"` 항목 추가 |

## 7. 이월 모달 (Carryover)

### 7.1 트리거

사이클 상세 페이지의 `[사이클 완료]` 버튼 클릭 시, 미완료 이슈가 있으면 이월 모달 표시. 미완료 이슈가 없으면 즉시 완료 확인 `AlertDialog` 표시.

### 7.2 미완료 이슈 판정

상태 카테고리가 `completed` 또는 `cancelled`가 **아닌** 이슈

### 7.3 레이아웃

```
┌── Dialog (w-[560px]) ──────────────────────────────────────────┐
│                                                                 │
│  사이클 완료                                            [X]     │
│  text-lg font-semibold                                          │
│                                                                 │
│  ───────────────────────────────────────────────────────────── │
│                                                                 │
│  미완료 이슈가 4개 있습니다. 이 이슈들을 어떻게 처리하시겠습니까?   │
│  text-sm text-muted-foreground                                  │
│                                                                 │
│  ┌─ 미완료 이슈 목록 (max-h-[240px] overflow-y-auto) ────────┐ │
│  │                                                            │ │
│  │  ● High   WRK-42   로그인 버그                      ● IP  │ │
│  │  ● Med    WRK-43   회원가입 API                     ● Todo │ │
│  │  ● Low    WRK-47   리팩토링                         ● BL   │ │
│  │  ● None   WRK-48   테스트 코드 추가                 ● Todo │ │
│  │                                                            │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                 │
│  이동 대상                                                      │
│  text-sm font-medium                                            │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Sprint 2 (Draft)                                   ▼   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ───────────────────────────────────────────────────────────── │
│                                                                 │
│                                   [취소]   [사이클 완료]        │
│                                   outline    primary            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7.4 컴포넌트 상세

#### 미완료 이슈 목록

| 속성 | 값 |
|------|-----|
| 항목 높이 | `h-9` (36px) |
| 항목 구조 | 우선순위 dot + 이슈 키 + 제목 (truncate) + 상태 뱃지 |
| 최대 높이 | `max-h-[240px] overflow-y-auto` |
| 스타일 | 읽기 전용 목록 (체크박스 없음) |

#### 이동 대상 드롭다운

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `Select` (shadcn/ui) |
| 옵션 | 같은 프로젝트의 다른 사이클 (Draft/Active) + `"사이클에서 제거 (백로그)"` |
| 기본값 | 다음 Draft/Active 사이클이 있으면 자동 선택, 없으면 `"사이클에서 제거"` |
| 옵션 표시 | `"{사이클명} ({상태})"` |
| `"사이클에서 제거"` 옵션 | `text-muted-foreground`, 구분선 위에 위치 |

### 7.5 키보드

| 키 | 동작 |
|----|------|
| `Esc` | 모달 닫기 (사이클 완료 취소) |
| `Enter` (버튼 포커스) | 사이클 완료 실행 |
| `Tab` | 드롭다운 → 취소 → 완료 순 이동 |

## 8. 사이클 뱃지 (이슈 속성)

### 8.1 이슈 상세에서 사이클 표시

이슈가 사이클에 속한 경우 이슈 상세 패널 속성 영역에 사이클 뱃지를 표시합니다.

```
┌─ 이슈 속성 영역 ──────────────────────────┐
│                                            │
│  상태        ● In Progress                 │
│  우선순위    🔴 Urgent                     │
│  담당자      👤 Luke Kim                   │
│  사이클      🔄 Sprint 1                   │  ← 사이클 뱃지
│  마감일      📅 04/15                      │
│                                            │
└────────────────────────────────────────────┘
```

#### 사이클 뱃지 스타일

| 속성 | 값 |
|------|-----|
| 아이콘 | `RefreshCw` (Lucide, 14px) |
| 텍스트 | 사이클 이름, `text-sm` |
| 배경 | `bg-muted` |
| 패딩 | `px-2 py-0.5` |
| 모서리 | `rounded-md` |
| 호버 | `hover:bg-accent cursor-pointer` |
| 클릭 | 해당 사이클 상세 페이지로 이동 |
| 미소속 | `"없음"` (`text-sm text-muted-foreground`) |

### 8.2 리스트 뷰에서 사이클 표시

리스트 뷰에서는 필터로 사이클별 이슈를 볼 수 있으며, 사이클 컬럼은 기본 비표시입니다. 사이클 상세 페이지 내 리스트에서는 사이클 컬럼이 불필요합니다.

## 11. 사이클 활성화 규칙

### 11.1 활성화 조건

| 조건 | 처리 |
|------|------|

| 이미 Active 사이클 존재 | `AlertDialog` — `"이미 활성 사이클이 있습니다. 기존 사이클을 먼저 완료해 주세요."` |

| Completed → 재활성화 | 불가 |

## 12. 접근성 (Accessibility)

### 12.1 ARIA

| 요소 | ARIA |
|------|------|
| 사이클 리스트 | `role="list" aria-label="사이클 목록"` |
| 사이클 카드 | `role="listitem" aria-label="{사이클명}, {상태}, 진행률 {N}%"` |
| 진행률 바 | `role="progressbar" aria-valuenow={percent} aria-valuemin={0} aria-valuemax={100} aria-label="사이클 진행률"` |
| 상태 뱃지 | `aria-label="{상태명}"` |
| 이슈 추가 팝오버 | `role="dialog" aria-label="이슈 추가"` |
| 이월 모달 | `role="dialog" aria-modal="true" aria-labelledby="carryover-title"` |

### 12.2 포커스 관리

- 사이클 생성 모달 열림: 이름 입력에 포커스
- 이슈 추가 팝오버 열림: 검색 입력에 포커스
- 이월 모달 열림: 드롭다운에 포커스
- 모달/팝오버 닫힘: 트리거 버튼으로 포커스 복귀
