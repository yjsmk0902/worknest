# Worknest Wiki 레이아웃 설계 (CP5-DS-1)

> 이 문서는 Wiki 사이드바 섹션, 페이지 트리, 스페이스 페이지, Breadcrumb, 스페이스 생성/설정, 빈 상태, 반응형 동작을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

Wiki는 워크스페이스 내 지식을 구조화하여 관리하는 모듈입니다. Wiki Space 안에 계층적 페이지 트리를 구성하며, 좌측 트리 패널과 우측 에디터 영역으로 나뉩니다.

### 1.1 핵심 원칙

- **계층적 구조**: 최대 6단계 깊이의 페이지 트리로 지식을 체계적으로 정리
- **빠른 탐색**: 사이드바에서 스페이스/페이지를 즉시 확인, Breadcrumb으로 위치 파악
- **직관적 관리**: 드래그 앤 드롭으로 페이지 이동, 우클릭 컨텍스트 메뉴로 빠른 작업
- **키보드 우선**: 트리 탐색, 페이지 생성/이동 모두 키보드로 가능

## 2. 사이드바 Wiki 섹션

### 2.1 기본 구조

```
│  ── Wiki ───────────  │  ← 섹션 라벨
│                       │     text-xs font-medium uppercase
│  📄  DevOps       ▶   │     tracking-wider text-muted-foreground
│  📄  Design       ▶   │
│  📄  Engineering  ▶   │
│  ＋  스페이스 추가      │
```

| 속성 | 값 |
|------|-----|
| 섹션 라벨 | `"Wiki"`, `text-xs font-medium uppercase tracking-wider text-muted-foreground` |
| 라벨 패딩 | `px-3 mb-1` |
| 항목 높이 | `h-8` (32px) |
| 항목 패딩 | `px-3` |
| 항목 텍스트 | `text-sm font-normal` |
| 아이콘 | `FileText` (Lucide, 16px) |
| 간격 | `gap-2` (아이콘-텍스트) |
| 호버 | `bg-accent rounded-md` |
| 활성 상태 | `bg-accent font-medium` |
| 확장 화살표 | `ChevronRight` (Lucide, 14px), 확장 시 90도 회전 (`transition-transform duration-150`) |
| 추가 버튼 | `text-muted-foreground hover:text-foreground`, `Plus` 아이콘 (16px) |

### 2.2 스페이스 확장 (페이지 트리)

```
│  📄  DevOps        ▼   │
│     📄 배포 가이드       │  ← indent 24px (레벨 0)
│       📄 AWS 설정       │  ← indent 40px (레벨 1)
│         📄 EC2 설정     │  ← indent 52px (레벨 2, 최대 인덴트)
│     📄 모니터링         │
│     ＋ 페이지 추가       │  ← indent 24px
```

#### 인덴트 규칙 (사이드바 내)

| 레벨 | padding-left | 비고 |
|------|-------------|------|
| 스페이스 | `8px` | 루트 |
| 레벨 0 (루트 페이지) | `24px` | +16px |
| 레벨 1 | `40px` | +16px |
| 레벨 2 | `52px` | +16px (최대, 이후 고정) |
| 레벨 3~5 | `52px` | 고정 (사이드바에서는 더 들여쓰지 않음) |

> 사이드바의 좁은 너비(240px)를 고려하여 인덴트는 52px에서 고정합니다. 전체 페이지 트리 패널에서는 6단계까지 표시합니다.

### 2.3 페이지 트리 항목 스타일

| 속성 | 값 |
|------|-----|
| 높이 | `h-7` (28px) — 사이드바 내에서는 컴팩트 |
| 아이콘 | `FileText` (Lucide, 14px) |
| 텍스트 | `text-sm truncate` |
| 호버 | `bg-accent rounded-sm` |
| 현재 페이지 | `bg-accent font-medium text-foreground` |
| 확장 토글 | `ChevronRight` (12px), 하위 페이지가 있을 때만 표시 |
| `＋ 페이지 추가` | 스페이스 내 마지막 항목, `text-xs text-muted-foreground hover:text-foreground` |

## 3. 페이지 트리 패널 (Wiki Space 내)

### 3.1 전체 레이아웃

```
┌── Sidebar ──┬── Page Tree ─────┬── Content Area ────────────────────┐
│             │                   │                                     │
│  (앱 셸     │  DevOps Space     │  ┌─ Breadcrumb ──────────────────┐ │
│   사이드바)  │  6 pages   ⚙     │  │ DevOps > 배포 가이드 > AWS 설정│ │
│             │                   │  └───────────────────────────────┘ │
│             │  ┌─ 트리 ──────┐  │                                     │
│             │  │ ▼ 배포 가이드│  │  ┌─ Editor ─────────────────────┐ │
│             │  │   ▶ AWS 설정│  │  │                               │ │
│             │  │   ▶ GCP 설정│  │  │  AWS EC2 설정 가이드           │ │
│             │  │ ▶ 모니터링  │  │  │  h1, font-bold                │ │
│             │  │ ▶ CI/CD    │  │  │                               │ │
│             │  │            │  │  │  (에디터 콘텐츠)                │ │
│             │  │            │  │  │                               │ │
│             │  │ ─ ─ ─ ─ ─ │  │  │                               │ │
│             │  │ ＋ 새 페이지│  │  │                               │ │
│             │  └────────────┘  │  └───────────────────────────────┘ │
│             │                   │                                     │
└─────────────┴───────────────────┴─────────────────────────────────────┘
```

### 3.2 페이지 트리 패널 스타일

| 속성 | 값 |
|------|-----|
| 너비 | `w-[240px]` |
| 배경 | `bg-muted/30` |
| 보더 | `border-r border-border` |
| 높이 | 전체 높이 (`h-full`) |
| 레이아웃 | `flex flex-col` |
| 오버플로 | `overflow-y-auto` (트리 영역만) |

#### 패널 헤더

| 속성 | 값 |
|------|-----|
| 높이 | `h-12` (48px) |
| 패딩 | `px-3` |
| 레이아웃 | `flex items-center justify-between` |
| 스페이스 이름 | `text-sm font-semibold` |
| 페이지 수 | `text-xs text-muted-foreground` |
| 설정 아이콘 | `Settings` (Lucide, 16px), `text-muted-foreground hover:text-foreground cursor-pointer` |
| 하단 보더 | `border-b border-border` |

### 3.3 트리 항목 상세

```
┌─ 트리 항목 ──────────────────────────────┐
│                                           │
│  [▶/▼ 12px] [📄 16px]  배포 가이드        │
│  chevron     icon       text-sm           │
│  (조건부)                truncate          │
│                                           │
└───────────────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 높이 | `h-8` (32px) |
| 레이아웃 | `flex items-center gap-1.5` |
| 인덴트 | 레벨 * 16px (`pl-[{level * 16}px]`) |
| 최대 인덴트 | 레벨 6 = 96px |
| Chevron | `ChevronRight` (12px), 하위 페이지 있을 때 표시, 확장 시 90도 회전 |
| 아이콘 | `FileText` (Lucide, 16px) |
| 텍스트 | `text-sm truncate` |
| 호버 | `bg-accent/50 rounded-sm` |
| 현재 페이지 | `bg-accent font-medium` |
| 포커스 | `ring-1 ring-primary/30 rounded-sm` |

#### 인덴트 상세 (페이지 트리 패널 내)

| 레벨 | padding-left | 비고 |
|------|-------------|------|
| 0 (루트 페이지) | `8px` | 기본 |
| 1 | `24px` | +16px |
| 2 | `40px` | +16px |
| 3 | `56px` | +16px |
| 4 | `72px` | +16px |
| 5 (최대) | `88px` | +16px |

### 3.4 드래그 앤 드롭 (dnd-kit)

#### DnD 피드백

| 상태 | 스타일 |
|------|--------|
| 드래그 중 항목 | `opacity-85 shadow-lg scale-[1.02] cursor-grabbing` |
| 원위치 placeholder | `border-2 border-dashed border-border bg-muted/30 rounded-sm h-8` |
| 드롭 위치 표시 (사이) | `border-t-2 border-primary` (2px 파란 선, 삽입 위치) |
| 드롭 위치 표시 (자식) | 대상 항목에 `ring-2 ring-primary/30 bg-primary/5` |

#### DnD 규칙

| 규칙 | 설명 |
|------|------|
| 순환 참조 방지 | 자기 자신 또는 하위 페이지로 이동 불가 |
| 최대 깊이 | 6단계 초과 시 드롭 불가 (드롭 존 비활성) |
| 순서 변경 | 같은 부모 내에서 위/아래 이동 (sort_order 업데이트) |
| 부모 변경 | 다른 페이지의 하위로 이동 (parent_id 업데이트) |
| 낙관적 업데이트 | 드롭 즉시 UI 반영, API 실패 시 롤백 + 토스트 |

### 3.5 `＋ 새 페이지` 버튼

| 속성 | 값 |
|------|-----|
| 위치 | 트리 하단 고정 (`mt-auto`) |
| 보더 | `border-t border-border` |
| 패딩 | `p-2` |
| 텍스트 | `"＋ 새 페이지"`, `text-sm text-muted-foreground hover:text-foreground` |
| 아이콘 | `Plus` (Lucide, 16px) |
| 동작 | 루트 레벨에 새 페이지 생성, 에디터에서 즉시 열기 |

### 3.6 우클릭 컨텍스트 메뉴

```
우클릭: 트리 항목
      │
      └─ ContextMenu (w-[200px])
         ┌──────────────────────────┐
         │  📄 새 하위 페이지          │
         │  ✏ 이름 변경               │
         │  📋 이동                   │  ← 서브메뉴: 스페이스/페이지 선택
         │  ⭐ 즐겨찾기 추가            │
         │  ────────────────────── │
         │  🗑 삭제                   │  ← text-destructive
         └──────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `ContextMenu` (shadcn/ui, Radix) |
| 너비 | `w-[200px]` |
| 항목 스타일 | `flex items-center gap-2 px-2 py-1.5 text-sm rounded-sm cursor-pointer` |
| 호버 | `bg-accent` |
| 삭제 확인 | `AlertDialog` — `"이 페이지를 삭제하시겠습니까? 하위 페이지도 함께 삭제됩니다."` |

#### 이름 변경 동작

```
트리 항목이 인라인 편집 모드로 전환:

│   [📄] [                     ] │  ← Input 활성화
│         ^^^^^^^^^^^^^^^^^^      │     기존 이름이 선택된 상태
│         auto-focus, 전체 선택   │
```

| 속성 | 값 |
|------|-----|
| 트리거 | 컨텍스트 메뉴 `"이름 변경"` 또는 `F2` |
| 컴포넌트 | 인라인 `Input`, 트리 항목 내에서 텍스트를 입력으로 교체 |
| 초기 상태 | 기존 이름이 전체 선택됨 |
| 확정 | `Enter` 또는 외부 클릭 (blur) |
| 취소 | `Esc` — 원래 이름으로 복귀 |
| 유효성 | 빈 값 불가 |

## 4. Breadcrumb 네비게이션

### 4.1 구조

```
DevOps  >  배포 가이드  >  AWS 설정
Space      Parent          Current Page
```

### 4.2 스타일

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `Breadcrumb` (shadcn/ui) |
| 위치 | 에디터 콘텐츠 영역 상단 |
| 패딩 | `px-6 py-2` (콘텐츠 영역과 동일 수평 정렬) |
| 구분자 | `ChevronRight` (Lucide, 14px), `text-muted-foreground` |
| 항목 텍스트 | `text-sm text-muted-foreground` |
| 현재 페이지 | `text-sm text-foreground font-medium` (링크 아님) |
| 링크 호버 | `hover:text-foreground` |
| 클릭 | 해당 스페이스/페이지로 이동 |
| 오버플로 | 경로가 길 경우 중간 항목을 `...` 으로 축약, 클릭 시 드롭다운으로 전체 경로 표시 |

### 4.3 Breadcrumb 오버플로

4단계 이상일 때:

```
DevOps  >  ...  >  EC2 설정  >  보안 그룹
                ↓ (클릭 시)
           ┌────────────────┐
           │ 배포 가이드     │
           │ AWS 설정       │
           └────────────────┘
```

| 속성 | 값 |
|------|-----|
| 축약 기준 | 항목 4개 이상 시 중간 항목 축약 |
| `...` | `MoreHorizontal` 아이콘 또는 `"..."` 텍스트, `cursor-pointer` |
| 드롭다운 | `DropdownMenu` (shadcn/ui), 축약된 항목 나열 |

## 5. Wiki Space 페이지

### 5.1 라우트

```
/:orgSlug/:wsSlug/wiki/:spaceSlug
```

### 5.2 레이아웃

Wiki Space 페이지는 스페이스를 처음 클릭했을 때 표시되는 랜딩 페이지입니다. 첫 번째 페이지가 있으면 자동으로 해당 페이지를 표시합니다. 페이지가 없으면 빈 상태를 표시합니다.

```
┌── Sidebar ──┬── Page Tree ─────┬── Content Area ────────────────────┐
│             │                   │                                     │
│             │  DevOps Space     │  ┌─ 빈 상태 (페이지 없을 때) ─────┐ │
│             │  0 pages   ⚙     │  │                                │ │
│             │                   │  │     [FileText 아이콘, 48px]    │ │
│             │  ┌─ 트리 ──────┐  │  │     text-muted-foreground     │ │
│             │  │             │  │  │                                │ │
│             │  │  (비어있음)  │  │  │  아직 페이지가 없습니다          │ │
│             │  │             │  │  │  text-lg font-medium           │ │
│             │  │             │  │  │                                │ │
│             │  │             │  │  │  첫 번째 페이지를 만들어         │ │
│             │  │ ─ ─ ─ ─ ─ │  │  │  지식을 정리하세요               │ │
│             │  │ ＋ 새 페이지│  │  │  text-sm text-muted-foreground │ │
│             │  │            │  │  │                                │ │
│             │  └────────────┘  │  │  [＋ 페이지 만들기] primary      │ │
│             │                   │  │                                │ │
│             │                   │  └────────────────────────────────┘ │
│             │                   │                                     │
└─────────────┴───────────────────┴─────────────────────────────────────┘
```

### 6.3 폼 필드 상세

#### 슬러그

| 속성 | 값 |
|------|-----|
| 컴포넌트 | `Input` (shadcn/ui) |

| 속성 | 값 |

| 백드롭 | `bg-black/30` (클릭 시 닫기) |
| 전환 | `transition-transform duration-150 ease-out` |
| 닫기 | 백드롭 클릭, `Esc`, 또는 페이지 선택 시 |

## 10. 키보드 내비게이션

### 10.1 페이지 트리 키보드

| 키 | 동작 |
|----|------|
| `↑` / `K` | 이전 항목으로 포커스 이동 |
| `↓` / `J` | 다음 항목으로 포커스 이동 |
| `→` | 펼치기 (접힌 항목) / 첫 번째 자식으로 이동 (펼친 항목) |
| `←` | 접기 (펼친 항목) / 부모로 이동 (접힌 항목) |
| `Enter` | 포커스된 페이지 열기 |
| `F2` | 이름 변경 모드 진입 |
| `Delete` / `Backspace` | 삭제 확인 다이얼로그 |
| `N` | 새 페이지 생성 (현재 포커스 항목의 형제로) |
| `Shift+N` | 새 하위 페이지 생성 (현재 포커스 항목의 자식으로) |

### 10.2 포커스 표시

포커스된 트리 항목에 `ring-1 ring-primary/30 rounded-sm` 아웃라인을 적용합니다.

## 11. 접근성 (Accessibility)

### 11.1 ARIA

| 요소 | ARIA |
|------|------|
| 페이지 트리 | `role="tree" aria-label="페이지 트리"` |
| 트리 항목 | `role="treeitem" aria-expanded={expanded} aria-level={level} aria-selected={isCurrent}` |
| 트리 그룹 | `role="group"` (자식 노드 래퍼) |
| Breadcrumb | `<nav aria-label="Breadcrumb">` + `<ol>` |
| 스페이스 설정 | `role="region" aria-label="스페이스 설정"` |
| 검색 | `role="search" aria-label="멤버 검색"` |

### 11.2 스크린 리더

- 트리 항목: `"{페이지명}, 레벨 {N}, {접힘/펼침}, {M}개 하위 페이지"` 알림
- 드래그 앤 드롭: `"{페이지명}을 {대상}의 하위로 이동했습니다"` 알림
- 페이지 생성: `"새 페이지가 생성되었습니다"` 알림
- 페이지 삭제: `"{페이지명}이 삭제되었습니다"` 알림
