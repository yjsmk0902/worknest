# Worknest 프로젝트 화면 설계 (CP2-DS-1)

> 이 문서는 프로젝트 리스트 페이지, 프로젝트 생성 모달, 프로젝트 설정 페이지의 레이아웃과 컴포넌트 스펙을 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

### 2.5 프로젝트 이름 입력

| 속성 | 값 |
|------|-----|
| 필수 | 예 |
| 최대 길이 | 50자 |
| 유효성 | 빈 값 불가 |
| 자동 Prefix | 이름 입력 시 접두사 자동 생성 (아래 규칙 참조) |
| 에러 | `"프로젝트 이름을 입력해주세요"` (`text-sm text-destructive`) |

### 2.6 접두사 (Prefix) 입력

#### 자동 생성 규칙

```
1. 영문 단어일 경우: 첫 글자들을 대문자로 조합
   "Worknest Backend" → "WB"
   "API Server" → "AS"

2. 단일 단어일 경우: 앞 3~4글자 대문자
   "Mobile" → "MOB"
   "Frontend" → "FE"

3. 한글일 경우: 영문 변환 없이 빈 값 → 수동 입력 유도
```

#### 수동 편집

- 사용자가 접두사 필드를 직접 수정하면 자동 생성 비활성화
- 이름 재입력 시에도 수동 입력값 유지 (사용자 의도 존중)
- 대문자 영문만 허용: `A-Z`, 2~5자
- 입력 시 자동 대문자 변환

#### 실시간 중복 체크

| 상태 | 표시 | 색상 |
|------|------|------|
| 체크 중 | `[스피너] 확인 중...` | `text-muted-foreground` |
| 사용 가능 | `✓ 사용 가능` | `text-green-600` (라이트) / `text-green-400` (다크) |
| 중복 | `✗ 이미 사용 중인 접두사입니다` | `text-destructive` |
| 형식 오류 | `✗ 영문 대문자 2~5자로 입력해주세요` | `text-destructive` |

```
중복 체크 타이밍: 300ms 디바운스
API: GET /api/projects/check-prefix?prefix={value}
```

### 2.7 모달 상태

| 상태 | 설명 |
|------|------|
| 기본 | 빈 폼, 아이콘 = 기본(Folder), 만들기 버튼 `disabled` |
| 유효 | 이름 입력 + 접두사 사용 가능 → 만들기 버튼 활성 |
| 생성 중 | `[스피너] 생성 중...`, 모든 필드 `disabled` |
| 성공 | 모달 닫힘 + 토스트: `"프로젝트가 생성되었습니다"` + 해당 프로젝트로 이동 |
| 실패 | 토스트: `"프로젝트 생성에 실패했습니다. 다시 시도해주세요."` |

### 2.8 키보드 인터랙션

| 키 | 동작 |
|------|------|
| `Tab` | 필드 간 이동 (아이콘 → 이름 → 접두사 → 설명 → 취소 → 만들기) |
| `Enter` | 만들기 버튼 활성 상태일 때 제출 |
| `Esc` | 모달 닫기 |

## 3. 프로젝트 설정 페이지

### 3.1 라우트

```
/:orgSlug/:wsSlug/projects/:prefix/settings           → 일반
/:orgSlug/:wsSlug/projects/:prefix/settings/members    → 멤버
/:orgSlug/:wsSlug/projects/:prefix/settings/labels     → 라벨
```

### 3.2 탭 구성

```
┌──────────────────┐
│  일반              │  ← /:orgSlug/:wsSlug/projects/:prefix/settings
│  멤버              │  ← /:orgSlug/:wsSlug/projects/:prefix/settings/members
│  라벨              │  ← /:orgSlug/:wsSlug/projects/:prefix/settings/labels
```

> Settings Nav 스타일은 `settings-pages.md` 섹션 1.1과 동일합니다.

## 4. 접근성 (Accessibility)

### 4.1 프로젝트 카드

- 카드 전체: `role="link"`, `tabindex="0"`, `aria-label="프로젝트: {이름}"`
- 키보드: `Enter` 또는 `Space`로 이동
- 포커스: `focus-visible:ring-2 ring-ring ring-offset-2`
- 그리드: `role="list"`, 각 카드 `role="listitem"`

### 4.2 모달

- `role="dialog" aria-modal="true" aria-labelledby="제목"`
- 포커스 트랩: 모달 내부에서만 Tab 순환
- 닫기 시 트리거 요소로 포커스 복귀

### 4.3 테이블

- `role="table"`, 헤더 `role="columnheader"`
- 역할 드롭다운: `aria-label="역할 변경"`
- 작업 메뉴: `aria-label="작업 메뉴"`, `aria-haspopup="true"`
