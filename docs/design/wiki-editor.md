# Worknest Wiki 에디터 설계 (CP5-DS-2)

> 이 문서는 Wiki 페이지 에디터의 레이아웃, 슬래시 명령어 메뉴, 이슈 링크, 자동 저장 인디케이터, 툴바를 정의합니다.
> 디자인 시스템 토큰은 `design-system.md`를 참조합니다.

---

## 1. 개요

Wiki 에디터는 TipTap (ProseMirror) 기반의 블록 에디터입니다. 슬래시 명령어로 다양한 블록을 삽입하고, 인라인 서식을 적용할 수 있습니다. MVP에서는 단일 사용자 편집 + 자동 저장, v1.0에서 CRDT 동시 편집을 지원합니다.

### 1.1 핵심 원칙

- **집중 모드**: 에디터 본문을 중앙 정렬하여 글쓰기에 집중
- **슬래시 명령어**: `/` 입력으로 모든 블록 타입에 빠르게 접근
- **자동 저장**: 2초 디바운스로 자동 저장, 상태를 시각적으로 표시
- **익숙한 서식**: 마크다운 단축키 + 리치 텍스트 툴바 병행

## 3. 슬래시 명령어 (/) 메뉴

### 3.1 트리거

에디터에서 빈 줄 시작 위치에 `/` 를 입력하면 슬래시 명령어 메뉴가 표시됩니다.

### 3.2 레이아웃

```
커서 위치
│
└─ 슬래시 메뉴 (floating)
   ┌── w-[280px] ────────────────────────────────────┐
   │                                                   │
   │  ┌─────────────────────────────────────────────┐ │
   │  │ 🔍 블록 검색...                               │ │  ← 필터 입력
   │  └─────────────────────────────────────────────┘ │
   │                                                   │
   │  텍스트                                           │  ← 카테고리 헤더
   │  text-xs font-medium text-muted-foreground        │     uppercase tracking-wider
   │                                                   │
   │  ┌─────────────────────────────────────────────┐ │
   │  │ [H1]  제목 1        큰 섹션 제목              │ │  ← 항목
   │  │ [H2]  제목 2        중간 섹션 제목             │ │     icon + name + desc
   │  │ [H3]  제목 3        작은 섹션 제목             │ │
   │  │ [¶]   본문          일반 텍스트                │ │
   │  └─────────────────────────────────────────────┘ │
   │                                                   │
   │  리스트                                           │
   │                                                   │
   │  ┌─────────────────────────────────────────────┐ │
   │  │ [•]   글머리 기호    순서 없는 리스트          │ │
   │  │ [1.]  번호 매기기    순서 있는 리스트          │ │
   │  │ [☐]   체크리스트     할 일 목록               │ │
   │  └─────────────────────────────────────────────┘ │
   │                                                   │
   │  미디어                                           │
   │                                                   │
   │  ┌─────────────────────────────────────────────┐ │
   │  │ [🖼]  이미지         이미지 업로드             │ │
   │  │ [📎]  파일 첨부      파일 업로드               │ │
   │  └─────────────────────────────────────────────┘ │
   │                                                   │
   │  고급                                             │
   │                                                   │
   │  ┌─────────────────────────────────────────────┐ │
   │  │ [<>]  코드 블록      구문 강조 코드           │ │
   │  │ [⊞]   테이블         행/열 테이블             │ │
   │  │ ["]   인용구         인용 텍스트               │ │
   │  │ [─]   구분선         수평 구분선               │ │
   │  └─────────────────────────────────────────────┘ │
   │                                                   │
   └───────────────────────────────────────────────────┘
```

### 3.5 키보드 인터랙션

| 키 | 동작 |
|----|------|
| `/` | 메뉴 열기 (빈 줄 시작에서) |
| `↑` / `↓` | 항목 이동 |
| `Enter` | 선택된 항목 삽입 |
| `Esc` | 메뉴 닫기 (입력한 `/` 유지) |
| 문자 입력 | 실시간 필터 (예: `/h` → 제목 1, 2, 3 표시) |
| `Backspace` (검색어 전부 삭제) | 메뉴 닫기 |

### 3.6 접근성

| 속성 | 값 |
|------|-----|
| 메뉴 | `role="listbox" aria-label="블록 삽입 메뉴"` |
| 항목 | `role="option" aria-selected={isActive}` |
| 검색 | `role="combobox" aria-expanded="true" aria-controls="slash-menu-list"` |
| 알림 | `aria-live="polite"` — `"{N}개 결과"` |

## 4. 이슈 링크

### 4.1 자동 변환

에디터에서 `WORK-142` 와 같은 이슈 키 패턴 (`/[A-Z]{2,5}-\d+/`) 을 입력하면 자동으로 클릭 가능한 링크 노드로 변환됩니다.

### 4.2 변환 전

```
... 이 버그는 WORK-142 에서 추적하고 있습니다.
                ^^^^^^^^
                일반 텍스트 (패턴 감지 전)
```

### 4.3 변환 후

```
... 이 버그는 [WORK-142] 에서 추적하고 있습니다.
              ^^^^^^^^^^
              이슈 링크 노드
```

### 4.4 이슈 링크 스타일

| 속성 | 값 |
|------|-----|
| 배경 | `bg-primary/10` |
| 텍스트 | `text-primary` |
| 모서리 | `rounded` |
| 패딩 | `px-1` |
| 커서 | `cursor-pointer` |
| 호버 | `bg-primary/20 underline` |
| 폰트 | `font-mono text-sm` |
| 클릭 | 해당 이슈 페이지로 네비게이션 (`/:org/:ws/projects/:prefix/issues/:key`) |
| 존재하지 않는 이슈 | 링크 스타일 유지, 클릭 시 toast `"이슈를 찾을 수 없습니다"` |

### 4.5 구현

| 속성 | 값 |
|------|-----|
| TipTap Extension | `InputRule` + `PasteRule` |
| 패턴 | `/([A-Z]{2,5}-\d+)/` |
| 노드 타입 | Custom `issueMention` Mark 또는 Inline Node |
| 데이터 속성 | `data-issue-key="{WORK-142}"` |
| 렌더링 | React NodeView 또는 Mark 렌더러 |

## 5. 자동 저장 인디케이터

### 5.1 위치

에디터 콘텐츠 영역의 우상단 (Breadcrumb과 같은 행, 또는 바로 아래)

### 5.2 상태별 표시

```
┌─ 저장됨 ───────────────────┐
│  ✓ 저장됨                   │  ← 기본 상태
│  CheckCircle + text         │
│  text-muted-foreground      │
└─────────────────────────────┘

┌─ 저장 중 ──────────────────┐
│  ◌ 저장 중...               │  ← 저장 진행 중
│  Loader2 (spin) + text      │
│  text-muted-foreground      │
└─────────────────────────────┘

┌─ 저장 실패 ────────────────┐
│  ⚠ 저장 실패   [재시도]      │  ← 에러 상태
│  AlertTriangle + text + btn │
│  text-destructive           │
└─────────────────────────────┘
```

### 5.3 스타일 상세

#### 공통

| 속성 | 값 |
|------|-----|
| 레이아웃 | `flex items-center gap-1.5` |
| 위치 | 절대 위치 `absolute top-2 right-6` 또는 Breadcrumb 행 우측 |
| 텍스트 크기 | `text-xs` |
| 전환 | `transition-opacity duration-150` |

#### 저장됨

| 속성 | 값 |
|------|-----|
| 아이콘 | `CheckCircle` (Lucide, 14px) |
| 색상 | `text-muted-foreground` |
| 텍스트 | `"저장됨"` |
| 자동 숨김 | 3초 후 `opacity-0` (호버 시 다시 표시) |

#### 저장 중

| 속성 | 값 |
|------|-----|
| 아이콘 | `Loader2` (Lucide, 14px), `animate-spin` |
| 색상 | `text-muted-foreground` |
| 텍스트 | `"저장 중..."` |

#### 저장 실패

| 속성 | 값 |
|------|-----|
| 아이콘 | `AlertTriangle` (Lucide, 14px) |
| 색상 | `text-destructive` |
| 텍스트 | `"저장 실패"` |
| 재시도 버튼 | `Button` variant=`ghost` size=`sm`, `text-xs text-destructive hover:text-destructive underline` |
| 지속 | 숨기지 않음 (사용자가 재시도하거나 저장 성공할 때까지) |

### 5.4 자동 저장 정책

| 속성 | 값 |
|------|-----|
| 디바운스 | 타이핑 중지 후 **2초** |
| 저장 API | `PATCH /wiki-pages/:id` (content 필드만) |
| 실패 시 | 재시도 버튼 표시, 5초 후 자동 재시도 1회 |
| 포커스 해제 시 | 즉시 저장 (blur 이벤트) |

## 6. 고정 툴바 (Fixed Toolbar)

### 6.1 레이아웃

```
┌─ 툴바 ──────────────────────────────────────────────────────────────┐
│                                                                      │
│  B  I  U  S  |  H1 H2 H3  |  • 1. ☐  |  "" ─ <> ⊞  |  🔗 @  🖼   │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 6.2 스타일

| 속성 | 값 |
|------|-----|
| 위치 | 에디터 상단 고정 (`sticky top-0 z-10`) |
| 배경 | `bg-background/95 backdrop-blur-sm` |
| 하단 보더 | `border-b border-border` |
| 패딩 | `px-4 py-1.5` |
| 레이아웃 | `flex items-center gap-0.5` |
| 구분선 | `w-px h-5 bg-border mx-1` (그룹 사이) |

### 6.3 툴바 버튼

| 속성 | 값 |
|------|-----|
| 크기 | `w-8 h-8` (32px, 최소 터치 타겟) |
| 아이콘 크기 | 16px |
| 모서리 | `rounded-md` |
| 기본 상태 | `text-muted-foreground` |
| 호버 | `bg-accent text-foreground` |
| 활성 (서식 적용됨) | `bg-accent text-foreground` |
| 비활성 | `opacity-50 cursor-not-allowed` |
| 포커스 | `focus-visible:ring-2 ring-ring ring-offset-2` |
| 전환 | `transition-colors duration-150` |
| 툴팁 | 각 버튼에 Tooltip — 이름 + 단축키 (예: `"굵게 (Cmd+B)"`) |

### 6.4 툴바 그룹 및 버튼

#### 그룹 1: 인라인 서식

## 9. 마크다운 단축키

에디터는 마크다운 문법을 자동으로 감지하여 변환합니다:

| 입력 | 변환 결과 |
|------|----------|
| `# ` | 제목 1 (H1) |
| `## ` | 제목 2 (H2) |
| `### ` | 제목 3 (H3) |
| `- ` 또는 `* ` | 글머리 기호 리스트 |
| `1. ` | 번호 매기기 리스트 |
| `[] ` 또는 `[ ] ` | 체크리스트 |
| `> ` | 인용구 |
| `---` | 구분선 |
| `` ``` `` | 코드 블록 |
| `**text**` | 굵게 |
| `*text*` | 기울임 |
| `` `code` `` | 인라인 코드 |
| `~~text~~` | 취소선 |

## 11. 접근성 (Accessibility)

### 11.1 ARIA

| 요소 | ARIA |
|------|------|
| 에디터 영역 | `role="textbox" aria-multiline="true" aria-label="페이지 내용 편집기"` |
| 툴바 | `role="toolbar" aria-label="서식 도구 모음" aria-orientation="horizontal"` |
| 툴바 버튼 | `role="button" aria-pressed={isActive} aria-label="{서식명}"` |
| 슬래시 메뉴 | `role="listbox" aria-label="블록 삽입 메뉴"` |
| 저장 상태 | `role="status" aria-live="polite"` |

### 11.2 키보드 접근성

- 툴바: `Tab`으로 진입, `←→`로 버튼 이동, `Enter/Space`로 활성화
- 에디터: 모든 서식 단축키 (`Cmd+B`, `Cmd+I` 등) 지원
- 포커스: 에디터 ↔ 툴바 간 `Tab` 이동
